## 1. 개발 결과물 개요

**청년원스톱**은 흩어진 청년 공공서비스 **(온통청년의 수천 개 정책 · 고용24(워크넷)의
공고 · LH·SH 청년주택 공고 · 청년도약계좌 · 내일채움공제)** 를 **"내 조건" 한 번
입력**으로 전부 매칭해주는 반응형 웹앱이다. 각 서비스는 정책 단위로 별도 사이트·
앱을 운영해 **청년이 직접 다섯 개 채널을 돌아야** 하는 문제를 해소한다.

국무조정실 청년정책조정위원회는 중앙·지자체 청년정책 수를 **수천 건 규모**로 공식
집계하고 있으며, 이를 총망라하는 포털 '온통청년'을 운영 중이다 [1]. 고용노동부는
2024년 '고용24'를 출범시켜 워크넷·국민취업지원 등 고용 서비스를 통합했다 [2].
그러나 청년 개인의 **나이·소득·지역·취업상태·학력**에 따라 "신청 가능한 것"만
선별하는 UX는 공공 영역에 부재하거나 제한적이다. 청년원스톱은 **단일 조건 입력 →
정책·일자리·주택·금융상품을 교차 매칭** 해 이 공백을 메운다.

**한 줄 요약**: "조건 한 번 입력, 정책·일자리·주거·금융 신청 가능한 모든 것을
자동으로 — 마감 캘린더와 AI 자기소개서 초안까지."

## 2. 개발 결과물의 기능 및 특징

### 2.1 서비스의 목적 및 배경

#### 문제 정의

- **청년 인구는 줄었지만 청년정책은 폭증했다.**
  통계청 장래인구추계에 따르면 2024년 만 19~34세 청년 인구는 약 **1,010만 명
  수준**으로 감소 추세이지만 [3], 동시에 중앙부처·지자체의 **청년정책은 수천 건**
  으로 증가했고 [1], 정책 인지율·도달율은 여전히 낮다. 국무조정실 「청년정책 인지도
  조사」는 청년 상당수가 **신청 가능한 정책을 '한 번도' 사용해 본 적 없다**고 응답한
  것을 반복적으로 보여준다 [4].

- **청년 실업·주거비 부담은 개선되지 않았다.**
  통계청 「경제활동인구조사 청년층 부가조사」에 따르면 청년층 취업 준비 기간·고용
  안정성 지표는 여전히 불안정하며 [5], 국토연구원·한국도시연구소는 수도권 청년
  1인 가구의 **소득 대비 주거비(RIR)** 가 국제 경고선인 30%를 초과하는 사례가
  적지 않다고 보고한다 [6]. 현재 LH·SH가 **청년매입임대·행복주택·청년전세임대** 등
  다채로운 공고를 내지만, **공고 주기·자격 요건이 복잡**해 청년은 자격 파악만으로
  한 주를 소진한다 [7].

- **고용24가 통합되었으나 여전히 서비스 단위 사일로가 존재한다.**
  고용노동부는 2024년 워크넷·직업훈련포털(HRD-Net)·국민취업지원·고용보험 등을
  **고용24**로 통합 개편했다 [2]. 그러나 청년 대상 **맞춤형 교차 탐색** (예:
  "국민취업지원 + 내일채움공제 + 청년도약계좌 동시 신청 가능한가")은 **사용자
  몫** 으로 남아 있다.

- **청년도약계좌·내일채움공제 등 금융 지원의 중복 가능성 판단이 어렵다.**
  청년도약계좌는 서민금융진흥원이 운영하며 월 70만 원 한도, 5년 만기 구조다 [8].
  내일채움공제·청년내일채움공제 등은 고용노동부·중소벤처기업부 주관으로 중복·대체
  관계가 복잡하다 [9]. 청년이 **중복 가능 여부**를 스스로 판단하려면 법령·시행령
  독해가 필요하다.

- **청년센터·상담사 인력이 부족하다.**
  전국 청년센터(지자체 운영)의 상담사는 **수백 명 수준**으로, 개별 청년에게 1:1
  맞춤 안내를 제공하기 어렵다 [10]. AI 매칭·초안 생성이 상담 인력을 보완해야 한다.

#### 해결 방향

1. **내 조건(나이·소득·지역·취업상태·학력·재직)** 1회 입력 → 신청 가능한 정책·공고
   자동 매칭.
2. **청년정책·일자리·주택·금융**을 한 화면에서 교차 탐색.
3. **마감 캘린더**로 놓치기 쉬운 신청 마감 사전 알림.
4. **AI 신청서 초안**: 공통 자기소개서·사유서를 정책별 양식으로 자동 변환.

### 2.2 서비스의 핵심 기술·기능·구성·특징

#### (1) 5대 핵심 화면

| # | 화면 | 핵심 기능 |
|:--:|---|---|
| ① | **홈 (내 매칭)** | "내 조건"에 맞는 정책·일자리·주택·금융 상품 Top 20 카드 + 추천 사유 1줄 |
| ② | **정책 탐색** | 온통청년 전체 정책을 카테고리(주거·일자리·금융·문화·복지) + 마감 임박순 정렬 |
| ③ | **일자리 · 훈련** | 고용24 공고 + 국민취업지원·내일배움카드 훈련과정 교차 표시 |
| ④ | **AI 신청서** | 공통 자기소개서 업로드 → LLM이 정책·공고별 양식으로 변환·요약 [11] |
| ⑤ | **마감 캘린더** | 매칭된 신청 마감을 캘린더로, D-7 / D-1 푸시·SMS 알림 |

#### (2) AI 기능 (공모전 가점 10점 대상)

- **조건 기반 매칭 랭킹 `/api/ai/match`**
  입력: 사용자 조건(표준화). 처리: 각 정책·공고의 자격 요건을 **자격 필터 규칙
  엔진**으로 1차 필터 → **"적합도 점수"** 를 계산(조건 충족도·지원 금액·선정 확률
  추정·마감 임박도). 피드백 로그가 축적되면 LambdaMART로 고도화.

- **AI 신청서 초안 생성 `/api/ai/draft`**
  LLM Tool-use [11]. 입력: 공통 자기소개서 + 지원하려는 정책. LLM은 다음 도구를
  필요 시 호출: `get_policy_detail(id)`, `get_form_template(id)`, `policy_law(id)`.
  공공포털에 공개된 양식을 참고하여 **"왜 나에게 필요한가"** 섹션을 정책별로
  재구성한다. 사용자 최종 검수 의무.

- **중복·대체 관계 판단** *(Phase 2)*
  LLM이 정책 법령·시행령을 RAG로 조회해 "청년도약계좌 + 내일채움공제 동시 가입
  가능 여부" 같은 질의에 **출처 문장 인용**으로 답한다.

#### (3) 시스템 구성

```
[Frontend: Next.js 16 App Router · TypeScript · Tailwind CSS v4]
          ↓
[Backend API Routes — 공공데이터 프록시 + 5분 TTL 캐시]
  ├─ /api/youth/policies?cond=...          (온통청년 정책 메타)
  ├─ /api/jobs?cond=...                    (고용24 공고)
  ├─ /api/housing?cond=...                 (LH·SH 청년주택 공고)
  ├─ /api/finance?cond=...                 (청년도약계좌·공제)
  ├─ /api/training?cond=...                (HRD-Net 훈련과정)
  ├─ /api/ai/match                         (매칭 랭킹)
  └─ /api/ai/draft                         (신청서 초안)
          ↓
[공공데이터포털 API] [LLM API + RAG (법령·시행령)] [Supabase DB]
```

- **자격 필터 규칙 엔진**: JSON Schema 기반, 각 정책 자격을 DSL로 선언.
- **정책 RAG**: 법제처 국가법령정보 API로 관련 법령·시행령 인덱싱 [12].

### 3.2 독창적·창의적 요소

1. **"내 조건 1회 입력 → 전 영역 동시 매칭"**
   청년이 반복 입력하는 **나이·소득·지역·취업상태·학력**을 한 번만 입력하면
   정책·일자리·주거·금융 4개 영역이 교차 매칭된다. 정부 디지털플랫폼 전략의
   "마이데이터 기반 원스톱" 방향과 정합 [13].

2. **정책 법령 RAG 기반 중복·대체 판단**
   기존 앱은 "나열"에 그치지만, 본 서비스는 법제처 법령·시행령 [12]을 RAG로 연결해
   "A + B 동시 가입 가능한가"를 **출처 문장 인용**으로 답한다. AI 환각을 구조적으로
   방지한다.

3. **AI 신청서 초안과 사용자 검수 워크플로**
   LLM이 대신 써주는 것이 아니라, **사용자의 공통 자소서를 정책별로 재구성**하는
   도구에 가깝다. 최종 검수를 사용자에게 맡겨 **개인정보·자기표현의 진정성**을
   보전한다.

4. **마감 캘린더 통합**
   청년이 가장 많이 놓치는 것은 "마감"이다. 매칭된 4종 영역을 **한 캘린더**로
   통합하고 D-7·D-1 알림을 제공. 이는 공공 서비스 접근성 연구 [4]가 공통적으로
   지적해온 문제를 직접 해소한다.

### 4.3 공공데이터 획득의 지속성

- 온통청년·고용24는 **정부 핵심 플랫폼**이며 공공데이터포털 상시 개방 대상 [1][2].
- **5분 TTL 캐시 + stale-while-revalidate** 로 단기 장애 강건.
- 정책·공고 데이터 갱신 주기가 상이하므로 **주기별 폴링 스케줄러**로 중복 호출 방지.

### 4.4 데이터 가공 및 결합

- 정책·공고·주택·금융을 **공통 스키마(영역·자격·금액·마감)** 로 정규화.
- 자격은 **JSON 규칙**으로 변환하여 사용자 조건과 Boolean 매칭.
- 모든 응답에 출처·원문 링크를 부착해 **사용자가 직접 재확인** 가능.

## 5. 개발 결과물의 기술성과 구현 및 실현가능성

### 5.1 활용한 주요 기술

| 레이어 | 기술 | 선정 사유 |
|---|---|---|
| 프레임워크 | Next.js 16 App Router + TypeScript | SSR, Vercel 즉시 배포 |
| 스타일 | Tailwind CSS v4 | 빠른 프로토타이핑 |
| 상태 관리 | TanStack Query + Zustand persist | 내 조건 영속화 |
| 차트 | Recharts | 마감·적합도 시각화 |
| AI | Claude Sonnet 4.6 (Tool-use + RAG) [11] | 긴 컨텍스트·한국어 정확도 |
| RAG | pgvector + 법령 문서 인덱스 | 정확한 근거 인용 |
| DB | Supabase (PostgreSQL) | 인증·RLS·실시간 |
| 배포 | Vercel | 무료 티어, 자동배포 |

### 5.2 구현 상태 (MVP 마일스톤)

- 5개 MVP 화면 와이어프레임 확정
- 공공데이터 5종 프록시 + AI 2종 + 법령 RAG 구조 확정
- 서울·부산 거주 청년 페르소나 3인 **시연 셋** 구축

**제출물**: Vercel 배포 URL + 접속 QR + 시연 영상.

### 5.3 실현 가능성 평가

## 6. 기대 효과

### 6.1 사회적 가치 창출

- **청년 정책 도달률 제고**
  현재 국무조정실 조사가 지속 지적해온 **"낮은 정책 인지·신청률"** [4]을 기술적으로
  돌파. 조건 기반 매칭이 **정책 인지 → 신청 → 선정**의 전환율을 끌어올린다.

- **청년 일자리 매칭 효율 개선**
  고용24 [2]의 공고와 훈련·국민취업지원을 **교차 탐색**함으로써 청년층 취업 준비
  기간의 단축을 기대할 수 있다. 경제활동인구조사 청년층 부가조사 [5]의 지표를
  정량 벤치마크로 삼는다.

- **청년 주거 안정 지원**
  LH·SH 청년주택 공고 [7]의 접근성을 개선해 수도권 청년의 **높은 RIR** [6] 부담
  완화에 기여.

- **공공 상담 인력 부담 경감**
  전국 청년센터 상담사 [10]의 반복 질의(자격 요건·중복 가능성)를 AI가 대신
  응답하면서 **상담 인력은 고난이도 1:1 사례에 집중** 가능.

### 6.2 상생협력·일자리 창출 효과

- **지자체 청년센터 B2G**: 상담사 대시보드 제공 (매칭 결과·신청 진행률).
- **대학·기업 B2B**: 대학교 진로센터·기업 인사팀에 청년 정책 매칭 모듈 제공.
- **청년 크리에이터**: "이 정책으로 집 구했어요" 콘텐츠 생태계.

### 6.3 정량 목표 (론칭 후 6~12개월)

| 지표 | 6개월 목표 | 12개월 목표 |
|---|---|---|
| 월 활성 이용자 (MAU) | 5만 | 30만 |
| 매칭된 정책·공고 신청 전환 | 1만 | 20만 |
| AI 신청서 초안 생성 | 3만 | 40만 |
| 마감 알림 유효 발송 | 10만 | 100만 |
| 평균 정책 인지 향상도 | +30% | +80% |

### 6.4 공공가치 선순환

> **사용자 매칭 로그 축적 → 정책 자격 규칙 정교화 →
> 지자체·부처에 "실제로 매칭된 사람이 적은 정책" 데이터 환류 → 정책 설계 개선**

청년원스톱의 데이터는 **정책 설계자**에게도 가치를 돌려준다.
