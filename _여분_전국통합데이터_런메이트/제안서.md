## 1. 개발 결과물 개요

**런메이트**는 대기질·기온·자외선 등 **환경 데이터**와 **공원·하천·둘레길**의 공간
데이터, 그리고 **공영자전거 대여소 실시간 재고**를 단일 의사결정 엔진에 결합해
시민 러너에게 **"오늘 여기서 이렇게 뛰자"** 를 한 장으로 추천하는 반응형 웹앱이다.

최근 한국 러닝 인구는 폭발적으로 증가해 **2023년 생활체육 참여자의 달리기 참여율은
역대 최고** 수준을 기록했으며 [1], 서울시 공공자전거 '따릉이'는 2024년 한 해에만
**4,385만 건** 이용, 10년 누적 **2.5억 건**을 돌파해 도시 단거리 이동 수단으로
자리잡았다 [2][3]. 그러나 러너들은 여전히 **"오늘 미세먼지 몇이야, 어디서 뛸까,
돌아올 때 어떻게 올까"** 를 여러 앱을 돌려가며 탐색하는 파편적 UX에 시달린다.
런메이트는 이 탐색을 **"신호등 카드 1장"** 으로 압축한다.

**한 줄 요약**: "대기질·날씨·공원·자전거 반납소까지 묶어, 오늘의 러닝 가능 여부와
코스를 1장으로 추천하는 시민 러너 동반 앱."

## 2. 개발 결과물의 기능 및 특징

### 2.1 서비스의 목적 및 배경

#### 문제 정의

- **러닝 인구는 사상 최대, 그러나 "뛸 수 있는 날"을 판단하기 어렵다.**
  문화체육관광부 「국민생활체육 참여실태조사」에 따르면 2023년 생활체육 참여율은
  역대 최고이며, 달리기(조깅·러닝)가 상위 종목으로 꾸준히 상승하고 있다 [1].
  그러나 이용자는 기상·대기질·자외선 정보를 **각각 다른 앱**에서 확인하고 공원
  POI와 교차 비교하는 수고를 반복한다.

- **미세먼지·폭염은 반복 경고되지만 판단 기준은 개인 몫이다.**
  에어코리아 통합대기환경지수(CAI)는 6단계 등급으로 정보를 제공하고 [4], 기상청은
  체감온도·자외선 지수를 시간별로 제공하지만 [5], 러너가 **"지금 30분 뛰어도
  괜찮은가"** 의 통합 결정을 이 데이터 위에서 **직관적으로** 내리기는 어렵다.

- **도시공원·하천변 러닝 코스 정보는 분산되어 있다.**
  국토교통부 도시공원 데이터셋에 따르면 전국 도시공원은 **2만여 개소**, 하천변
  자전거·보행 겸용 도로도 상당 수준으로 확보되어 있으나 [6], 러너 관점의 **거리별
  추천 코스**가 공공 채널에서 통합 제공되지 않는다.

- **공영자전거는 "러닝 후 편도 귀가" 수요와 자연스럽게 결합된다.**
  공영자전거 API(따릉이·타슈·어울링 등)는 **전국 통합 개방**되어 있고 [7],
  서울 따릉이의 10년 누적 2.5억 건 이용 [2][3]이 그 생활 밀착도를 입증한다. 러너는
  **출발점 → 목적지 편도 러닝 후 자전거로 귀가** 하는 패턴을 선호하지만, 현재는
  이를 묶는 앱이 없다.

- **시민 러닝 크루 문화의 확산.**
  한강 러닝 크루, 지역 러닝 동호회가 증가하며 **"함께 뛰자"** 수요가 커졌다. 이들은
  "다음 주 토 오전 여의도에서 8km, 페이스 6:00, 대기질 노란색 이상이면 취소" 같은
  **조건부 이벤트 조율 도구**를 필요로 한다.

#### 해결 방향

1. **대기질·체감온도·자외선 + 공원·하천·둘레길 + 공영자전거 반납소**를 단일
   의사결정 레이어로 통합.
2. **"러닝 신호등"** 카드가 🟢/🟡/🔴 + 이유 1줄로 결정 지원.
3. **페이스·거리별 AI 코스 추천**이 초·중·고급 러너별 최적 루트를 선택.
4. **"러닝 후 자전거 귀가"** 편도 코스 + 반납소 자동 매칭.

### 2.2 서비스의 핵심 기술·기능·구성·특징

#### (1) 5대 핵심 화면

| # | 화면 | 핵심 기능 |
|:--:|---|---|
| ① | **홈 (러닝 신호등)** | 🟢 / 🟡 / 🔴 + 이유 1줄 + 추천 시간대 3블록 (아침/저녁/야간) |
| ② | **코스 지도** | 공원·하천·둘레길 POI + 추천 3km/5km/10km 루트 + 반납소 마커 |
| ③ | **AI 코스 빌더** | 자연어 "조용한 5km 평지, 물 있는 곳" 등 복합 조건을 Tool-use로 해석 [8] |
| ④ | **러닝 기록** | GPS 트래킹(페이스·거리·고도) + 체크인, PR(Personal Record) 자동 갱신 |
| ⑤ | **러닝 크루** | 이벤트 생성/참여, 신호등 조건부 자동 취소·변경 알림 |

#### (2) AI 기능 (공모전 가점 10점 대상)

- **"러닝 신호등" 결정 모델 `/api/ai/signal`**
  입력: 대기질 등급·체감온도·자외선·습도·최근 강수·시간대·개인 신체조건(옵션).
  처리: 의료 가이드 라인(WHO 야외활동 권고 [9])과 스포츠 의학의 열지수 기준을
  조합한 **가중치 규칙 + 분류기**로 🟢/🟡/🔴을 결정한다. 개인화 피드백이 축적되면
  LightGBM으로 전환.

- **페이스·거리별 AI 코스 추천 `/api/ai/course`**
  LLM Tool-use [8]. LLM은 다음 도구를 호출한다:
  `nearby_parks(lat, lng)`, `nearby_trails(lat, lng, distance)`,
  `elevation_profile(route)`, `nearby_bikes(lat, lng)`, `air_quality_now()`,
  `weather_now()`. "강변 평지 5km, 끝에 반납소" 같은 복합 조건을 실시간 데이터로
  해석해 실사실 기반 코스를 반환한다.

- **러닝 크루 "스마트 취소"** *(Phase 2)*
  이벤트 조건(예: "미세먼지 나쁨이면 취소")을 LLM이 파싱하고 Cron으로 신호등
  상태 변화 시 자동 취소·재공지.

#### (3) 시스템 구성

```
[Frontend: Next.js 16 App Router · TypeScript · Tailwind CSS v4]
          ↓
[Backend API Routes — 공공데이터 프록시 + 5분 TTL 캐시]
  ├─ /api/aq?lat,lng                       (에어코리아 통합대기환경지수)
  ├─ /api/weather?lat,lng                  (기상청 초단기·단기 + 체감온도·자외선)
  ├─ /api/parks?near=lat,lng               (도시공원 + 하천 자전거길)
  ├─ /api/bikes?near=lat,lng               (전국 공영자전거 실시간 재고)
  ├─ /api/ai/signal                        (러닝 신호등)
  └─ /api/ai/course                        (AI 코스 빌더)
          ↓
[공공데이터포털 API] [LLM API] [OSM 경로 API]
```

- **경로 고도 프로파일**: OSM + SRTM DEM(공개 고도 데이터)으로 고도차 계산 → 평지/
  오르막 분류.
- **GPS 트래킹**: 브라우저 Geolocation + 이동 평활화.
- **오프라인 폴백**: 달리기 중 셀룰러 유실 대비 지도 타일·경로 로컬 캐시.

## 3. 기존 서비스와 차별성 및 독창성

### 3.1 기존 유사 서비스 비교

| 항목 | 민간 러닝 앱 | 공공 대기질 앱 | 공영자전거 앱 | **런메이트** |
|---|:---:|:---:|:---:|:---:|
| 대기질 기반 "러닝 가능" 신호 | ❌ | ⚠️ (일반용) | ❌ | **✅ (러너 특화)** |
| 공원·하천 코스 공공 POI | ❌ | ❌ | ❌ | **✅ (전국 통합)** |
| 공영자전거 귀가 편도 모드 | ❌ | ❌ | ✅ (단독) | **✅ (러닝과 결합)** |
| 자연어 코스 빌더 | ❌ | ❌ | ❌ | **✅ (LLM)** |
| 러닝 크루 조건부 취소 | ❌ | ❌ | ❌ | **✅** |
| 이용료 | 부분 유료 | 무료 | 저가 | **무료** |

### 3.2 독창적·창의적 요소

1. **러닝 결정을 "신호등"으로 압축**
   6단계 대기질·체감온도·자외선·습도를 조합한 **3단계 신호등 + 이유 1줄**이라는
   과감한 UX 압축. 의사결정 피로를 최소화해 **"실제로 뛰러 나가는" 전환율**을
   높인다. WHO 야외활동 권고의 기준선 [9]을 준수한다.

2. **러닝 + 공영자전거 편도 체인**
   "뛰기만" 하는 앱, "자전거만" 하는 앱은 많지만, **편도 러닝 후 반납소로 귀가**를
   하나의 플로우로 엮은 사례는 공공 영역에 부재하다. 서울 따릉이 10년 2.5억 건 [3]
   이용 생태계와 러닝 인구 [1]의 교집합을 겨냥한다.

3. **러닝 크루 "조건부 자동 취소"**
   러닝 크루 이벤트에 "미세먼지 나쁨 시 취소" 같은 조건을 걸면 LLM이 상태 변화에
   반응해 **자동 재공지**한다. 커뮤니티 운영 비용을 공공데이터가 대신 지불한다.

4. **탄소저감 리워드 연계**
   러닝은 자체로 저탄소 이동이 아니나, **러닝 + 자전거 귀가 = 단거리 자동차 대체**의
   효과가 있다. 탄소중립위원회 친환경 이동 장려 정책 [10]과 정합.

### 3.3 지식재산권 전략

- "환경·기상·공영자원 통합 러닝 의사결정 시스템 및 방법" BM 특허 출원 검토.
- 프록시·캐시 레이어 MIT 라이선스 공개.

## 4. 공공데이터 등 데이터 활용성

### 4.1 필수 활용 공공데이터

| 데이터명 | 원천 기관 | 획득 방법 | 서비스 내 활용 |
|---|---|---|---|
| **전국 공영자전거 실시간 정보** | 행정안전부·한국지역정보개발원 (11개 지자체 통합) [7] | 공공데이터포털 REST | 반납소 마커, 편도 귀가 매칭 |
| **에어코리아 통합대기환경지수** | 한국환경공단 [4] | Open API 1시간 주기 | 신호등 입력 |
| **기상청 동네예보·초단기** | 기상청 API허브 [5] | REST 1시간 주기 | 체감온도·자외선·강수 |
| **도시공원·하천 자전거길** | 국토교통부·지자체 [6] | 공공데이터포털 CSV/API | 코스 POI |
| **고도 데이터 (DEM)** | 국토지리정보원 [11] | 공개 래스터 | 평지/오르막 분류 |

### 4.2 타 기관 공공·민간 데이터 보조 활용

| 데이터 | 출처 | 활용 목적 |
|---|---|---|
| 공중화장실·음수대 | 행정안전부 | 코스 중간 시설 |
| 자외선·열지수 가이드 | WHO / 질병관리청 | 신호등 임계치 기준 |

### 4.3 공공데이터 획득의 지속성

- 공영자전거 통합 API는 **국가중점데이터**로 지속성이 제도적으로 보장된다 [12].
- **5분 TTL 캐시 + stale-while-revalidate**로 상류 API 단기 장애에 강건.
- 갱신 주기(대기질 1시간, 기상 1시간, 자전거 2분)를 존중해 Rate Limit 초과 방지.

### 4.4 데이터 가공 및 결합

- 공원·하천·자전거 반납소는 **위경도 공통 키**로 결합.
- 경로는 OSM 그래프 + SRTM 고도로 **거리 × 고도 × 포장면** 3차원 속성 벡터 구성.
- 사용자 러닝 로그는 **시간대별 혼잡 학습셋**으로도 활용 (Phase 2).

## 5. 개발 결과물의 기술성과 구현 및 실현가능성

### 5.1 활용한 주요 기술

| 레이어 | 기술 | 선정 사유 |
|---|---|---|
| 프레임워크 | Next.js 16 App Router + TypeScript | SSR, Vercel 즉시 배포 |
| 스타일 | Tailwind CSS v4 | 빠른 프로토타이핑 |
| 지도·경로 | Leaflet + OpenStreetMap + OSRM | 무료, API 키 불필요 |
| 상태 관리 | TanStack Query + Zustand persist | 러닝 로그 영속 |
| 차트 | Recharts | 페이스·고도 그래프 |
| AI | Anthropic Claude Haiku 4.5 (Tool-use) | 저비용, 구조화된 함수 호출 |
| GPS | Geolocation API + Kalman 평활화 | 모바일 브라우저 표준 |
| 배포 | Vercel | 무료 티어, 자동배포 |

### 5.2 구현 상태 (MVP 마일스톤)

- 5개 MVP 화면 와이어프레임 확정
- 공공데이터 4종 프록시 + AI 2종 엔드포인트 구조 확정
- 서울 한강공원·양재천·북한산 둘레길 **시연 셋** 준비

**제출물**: Vercel 배포 URL + 접속 QR + 러닝 시연 영상.

### 5.3 실현 가능성 평가

| 리스크 | 영향도 | 대응 |
|---|:---:|---|
| GPS 정확도 편차 | 중간 | Kalman 필터, 5초 평균, 터널·고층 구간 보정 |
| 공공 API 일시 장애 | 높음 | 5분 TTL 캐시 + stale-while-revalidate |
| 대기질 국지적 오차 | 중간 | 최근접 측정소 2개 가중 평균 |
| 개인정보 (러닝 경로) | 높음 | 기본 비공개, 공유 시 시작/종료점 블러 옵션 |
| AI 비용 | 낮음 | Claude Haiku + 캐싱, 월 $10 이하 |

### 5.4 향후 고도화 로드맵

| Phase | 기간 | 핵심 작업 |
|:--:|---|---|
| **Phase 1** | 출시 후 3개월 | 개인 러닝 로그 → 신호등 개인화 (민감군/체질 가중) |
| **Phase 2** | 4~6개월 | 러닝 크루 조건부 이벤트, 코치 챗봇 |
| **Phase 3** | 7~12개월 | 지자체 생활체육·마라톤 이벤트 연계, 탄소저감 포인트 |
| **Phase 4** | 12개월 이후 | 웨어러블 연동(HRM), 러닝 건강 리포트 |

## 6. 기대 효과

### 6.1 사회적 가치 창출

- **국민 생활체육 참여 증대**
  달리기는 생활체육 대표 종목 [1]. "뛸 수 있는가" 결정 비용을 낮추면 주 1회 이상
  러닝 인구가 늘어날 것으로 기대. 보수 가정으로 MAU 10만 명 × 월 2회 추가 러닝
  시 **연 240만 건** 의 신규 운동 세션이 유도된다.

- **기후·대기오염 건강 피해 예방**
  WHO는 야외활동 시 대기질 모니터링 중요성을 강조한다 [9]. 러너가 **고농도 미세먼지
  시 자동 경고**를 받음으로써 천식·심혈관 위험군의 노출이 감소한다. 한국환경공단
  통합대기환경지수 [4]를 러닝에 특화해 활용한다.

- **공영자전거 이용 유도 → 단거리 자동차 대체**
  서울 따릉이 10년 2.5억 건 [3]의 이용 패턴이 증명하듯, 공영자전거는 도시 단거리
  이동의 생활형 대안. 러닝 후 귀가 동선을 자전거로 유도함으로써 **단거리 자동차
  통행 감소** → 도심 탄소배출 저감 [10]에 기여.

- **공원·하천 공공공간의 활용 제고**
  도시공원 2만여 개소 [6]와 하천 자전거길의 비공식적 러닝 이용을 **가시화·통계화**
  하여, 지자체 공간 정비의 데이터 근거로 제공.

### 6.2 상생협력·일자리 창출 효과

- **생활체육회·지자체 확산**: 지역 마라톤·러닝 이벤트 등록 모듈 제공 → 지자체 확산
  용역 수요.
- **러닝 크리에이터·러닝 크루 B2C**: 앱에서 생성된 "오늘 코스"를 콘텐츠화하여
  플랫폼 성장.
- **스포츠 의학·헬스케어 B2B**: 개인 러닝 로그 기반 건강 리포트 → 보험·헬스케어
  제휴.

### 6.3 정량 목표 (론칭 후 6~12개월)

| 지표 | 6개월 목표 | 12개월 목표 |
|---|---|---|
| 월 활성 러너 (MAU) | 3만 | 15만 |
| 러닝 세션 체크인 | 10만 | 120만 |
| 공영자전거 귀가 전환 | 2만 | 30만 |
| 러닝 크루 이벤트 생성 | 1천 | 2만 |
| 신호등 추천 적중률 (실제 러닝 전환) | 30% | 50% |

### 6.4 공공가치 선순환

> **러닝 로그 축적 → 시간대·장소별 혼잡·공기질 매핑 →
> 지자체 공원 관리 데이터 → 시민 건강 증진**

이 선순환은 **스포츠-환경-교통**을 아우르는 공공데이터 통합 활용의 모범이 된다.
