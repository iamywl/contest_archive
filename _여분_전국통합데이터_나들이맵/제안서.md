## 1. 개발 결과물 개요

**나들이맵**은 영유아를 동반한 가족 외출에 필수적인 **공영주차장(실시간 잔여면수)·
공중화장실(기저귀교환대·장애인화장실 속성)·수유실·어린이도서관·공원**을 단일 지도에
교차 배치하고, 도착시간·혼잡도·유아 친화 속성을 기반으로 **"주차 → 화장실 → 놀이 →
식사 → 복귀"** 동선을 자동 생성하는 반응형 웹앱이다.

통계청 인구동향조사에 따르면 2023년 합계출산율은 **0.72**로 OECD 최저를 경신했으나
[1], 동시에 0~6세 영유아 인구는 여전히 **약 210만 명**이며, 이들을 돌보는 부모·
조부모 세대의 **외출 의사결정 피로도**가 개별 앱 다수 설치·정보 단절로 극심하다.
나들이맵은 이 파편화를 한 화면으로 해소한다. **"유아 동반" 한 번의 필터**로 모든
POI가 재정렬되어, 부모의 인지 부담을 최소화한다.

**한 줄 요약**: "아이 데리고 나가기 전 10분 — 나들이맵이 동선을 대신 그려줍니다."

## 2. 개발 결과물의 기능 및 특징

### 2.1 서비스의 목적 및 배경

#### 문제 정의

- **저출생 시대, 아이와 함께 나가기가 더 어려워졌다.**
  2023년 출생아 수는 23만 명으로 역대 최저, 합계출산율 0.72 [1]. 아이 수는 줄었지만
  **아이 한 명에게 투입되는 부모의 시간·정보 부담은 오히려 증가**했다. 여성가족부
  「2024 가족실태조사」는 영유아 동반 외출의 최대 애로로 "주차·화장실·수유실 탐색
  피로"를 지목한다 [2].

- **공공 유아 편의시설은 '지도에 없다'.**
  행정안전부·지자체가 전국 **공중화장실 약 8만 5천여 개소**를 관리하고 있고 [3],
  여성가족부는 전국 공공기관·문화시설에 **수유실 약 3,100여 개소**를 지정·운영한다
  [4]. 그러나 각각의 **기저귀교환대·유아용 좌변기·남녀 공용 수유실** 속성 필터가
  일반 지도앱에서 제공되지 않아, **"있어도 못 찾는" 구조적 단절**이 반복된다.

- **공영주차장 실시간 정보는 개방되었으나 묶이지 않았다.**
  행정안전부·한국지역정보개발원은 2025년 공영주차장 실시간 잔여면수 API를 전국
  통합 개방했다 [5]. 그러나 이 데이터는 **"목적지 주변 + 유아 편의"** 와 결합되지
  않은 채 각 지자체 사이트에 방치되어 있다.

- **어린이도서관·공원은 주말 집중 혼잡, 예측 부재.**
  문화체육관광부 「전국도서관통계」 기준 어린이도서관·작은도서관은 **1,300개관 이상**
  이 운영 중이며 [6], 주말 오후 1~3시 혼잡 피크가 뚜렷하다. 현 체계는 **"지금 갔을
  때 자리가 있는가"** 에 대한 실시간 정보를 제공하지 못한다.

- **부모의 외출 의사결정은 "10분 안에" 이루어진다.**
  한국갤럽 「영유아 부모 여가활동 조사」(2024) 기준 부모는 외출 결정·장소 탐색·짐
  패킹을 합쳐 평균 30분 이내에 처리해야 하며, 이 시간이 넘어가면 **외출 자체를 포기**
  하는 비율이 40%를 넘는다 [7]. 결국 정보 단절이 **가족 여가 기회의 단절**로 직결된다.

#### 해결 방향

1. **공영주차장 + 공중화장실 + 수유실 + 어린이도서관 + 공원**을 단일 지도에 통합.
2. **"유아 동반" 원클릭 필터**가 모든 POI의 유아 친화 속성을 재정렬.
3. **"오늘 나들이" AI 플래너**가 목적지 입력만으로 5단계 동선을 자동 생성.
4. **혼잡도 단기 예측**으로 "지금 가면 자리 있을까"에 선제적으로 답한다.

### 2.2 서비스의 핵심 기술·기능·구성·특징

#### (1) 5대 핵심 화면

| # | 화면 | 핵심 기능 |
|:--:|---|---|
| ① | **홈 (오늘 나들이)** | 목적지 입력 → AI가 주차장·화장실·식사·복귀 5단계 카드 자동 생성 |
| ② | **지도** | 반경 2km 내 주차장(잔여면수 3단계 색상) + 화장실(🍼 기저귀교환) + 수유실(🤱) + 어린이도서관(📚) + 공원(🌳) |
| ③ | **유아 필터** | 한 번의 토글로 "유아동반 OK" 속성만 표시 + 장애인 가족 모드 (휠체어·엘리베이터 경로) |
| ④ | **혼잡 예측** | 어린이도서관·공원의 현 시점 + 1·2시간 후 혼잡도 막대 (AI 예측) |
| ⑤ | **나들이 일기** | 체크인 기록 + 사진·메모 자동 정리, 공유 가능한 "오늘의 코스" 카드 생성 |

#### (2) AI 기능 (공모전 가점 10점 대상)

- **"오늘 나들이" 동선 자동 생성 `/api/ai/plan`**
  입력: 출발지·목적지·아이 수·연령대·동반(조부모/장애인)·선호(실내/실외).
  처리: LLM Tool-use로 주차장·화장실·수유실·도서관·공원을 순서대로 조회하여
  거리·혼잡도·편의 속성을 결합한 5단계 카드를 반환한다 [8]. Tool 목록:
  `parking_live(near)`, `toilets_with_babychange(near)`, `nursing_rooms(near)`,
  `kids_library(near)`, `parks(near)`, `weather_now()`.

- **혼잡도 단기 예측 `/api/ai/forecast`**
  초기에는 **요일·시간·날씨 피처**를 입력으로 하는 휴리스틱 + 이동평균을 사용하고,
  체크인 로그가 축적되면 Prophet·LightGBM 회귀로 전환한다. 응급실·교통 분야에서
  검증된 시계열 예측 패러다임을 **공공 육아 인프라**에 적용한다 [9][10].

- **주간 나들이 회고** *(Phase 2)*
  LLM이 "이번 주 도서관 2회, 공원 1회 — 지난 주 대비 야외시간 35분 증가"와
  같은 가족 공유용 카드를 자동 생성한다.

#### (3) 시스템 구성

```
[Frontend: Next.js 16 App Router · TypeScript · Tailwind CSS v4]
          ↓
[Backend API Routes — 공공데이터 프록시 + 5분 TTL 캐시]
  ├─ /api/parking?near=lat,lng             (공영주차장 실시간 잔여면수)
  ├─ /api/toilets?near=lat,lng&babyChange=1(공중화장실 + 기저귀교환대 필터)
  ├─ /api/nursing?near=lat,lng             (수유실)
  ├─ /api/kidslib?near=lat,lng             (어린이도서관)
  ├─ /api/parks?near=lat,lng               (도시공원·놀이터)
  ├─ /api/ai/plan                          (5단계 동선 생성)
  └─ /api/ai/forecast?poi=id               (혼잡도 예측)
          ↓
[공공데이터포털 API] [LLM API] [기상청 API허브]
```

- **공공 API 프록시**: 인증키 서버 보관, 5분 TTL + stale-while-revalidate.
- **지도**: Leaflet + OpenStreetMap. 유아 친화 속성은 **아이콘 + 색상 이중 인코딩**.
- **혼잡 예측 학습셋**: 체크인 로그를 시간축으로 누적, 5분 단위 집계.

### 3.2 독창적·창의적 요소

1. **원클릭 "유아 동반" 필터**
   모든 POI가 한 번의 필터로 재정렬된다. 단순 보기 전환이 아니라, 각 POI에 부착된
   **기저귀교환대·유아좌변기·수유 속성·유모차 접근성**을 데이터 레이어에서 결합한다.
   행정안전부 공중화장실 공공데이터의 세부 속성과 여성가족부 수유실 DB를 결합해
   구현한다 [3][4].

2. **"오늘 나들이" 5단계 동선 자동 생성**
   단일 POI 검색을 넘어, **주차 → 화장실 → 놀이 → 식사 → 복귀**를 시공간 체인으로
   묶어 1회 외출을 하나의 플로우로 완결한다. 국가 MaaS(K-MaaS) 전략이 교통수단 간
   연계에 주력하는 것과 달리 [11], **육아 편의 자원까지 MaaS 대상**으로 확장.

3. **혼잡도 단기 예측과 AI 플래너 결합**
   "지금 여기 자리 있는가"에 대한 실시간 정보 + **1~2시간 후 예측**을 결합해
   **외출 의사결정의 불확실성**을 제거한다. 부모의 평균 의사결정 시간(30분 [7])
   안에 **후보 2~3곳의 혼잡 트렌드**를 함께 제시한다.

4. **"장애인 가족" 모드**
   장애인 자녀 또는 동반 가족이 있는 경우, 기저귀교환대 + **장애인화장실 +
   엘리베이터 + 경사로** 까지 결합 필터. 이는 공고가 중시하는 "교통약자 이동지원"
   정책 방향과 정합한다 [5].

### 3.3 지식재산권 전략

- "유아 동반 가족 외출 자원 통합 동선 생성 시스템 및 방법" BM 특허 출원 검토.
- 프록시·캐시 레이어 MIT 오픈소스 공개 → 지자체·관광재단 재활용 유도.

## 4. 공공데이터 등 데이터 활용성

### 4.1 필수 활용 공공데이터

모두 **공공데이터포털(data.go.kr)** 에서 오픈 API로 제공되며, 2025년 행정안전부·
한국지역정보개발원 전국 통합 개방 대상 데이터 다수를 포함한다 [5].

| 데이터명 | 원천 기관 | 획득 방법 | 서비스 내 활용 |
|---|---|---|---|
| **공영주차장 실시간 잔여면수** | 행정안전부·지자체 | 통합 API, 5분 주기 | 지도 마커(3단계 색상), 동선 1단계 |
| **공중화장실 현황 (속성 포함)** | 행정안전부·지자체 | 공공데이터포털 [3] | 기저귀교환대·유아좌변기 필터 |
| **수유실 현황** | 여성가족부 [4] | 공공데이터포털 CSV/API | 유아 필터 필수 POI |
| **전국 어린이도서관·작은도서관** | 문화체육관광부 [6] | 도서관 정보나루 [12] | 혼잡 예측 대상 |
| **도시공원 정보** | 국토교통부·한국토지주택공사 | 공공데이터포털 | 놀이 단계 POI |

### 4.2 타 기관 공공·민간 데이터 보조 활용

| 데이터 | 출처 | 활용 목적 |
|---|---|---|
| 단기 기상·자외선 예보 | 기상청 API허브 [13] | 실내/실외 추천 전환 |
| 대기질 예보 | 에어코리아 [14] | 유아 외출 권고 |
| 지도 타일 | OpenStreetMap | 지도 시각화 |

### 4.3 공공데이터 획득의 지속성

- 활용 API는 공공데이터포털의 상시 개방 데이터이며 일부는 **국가중점데이터**로
  지속성이 제도적으로 보장된다 [15].
- **5분 TTL 캐시 + stale-while-revalidate**로 상류 API 단기 장애에 강건.
- 수유실·화장실 데이터는 갱신 주기가 긴 만큼 **월 1회 정기 동기화** 스케줄러로
  최신성을 유지.

### 4.4 데이터 가공 및 결합

- POI는 **위경도 공통 키**로 결합. Haversine 거리 + 5단계 동선 그래프 탐색.
- 혼잡 예측은 체크인 로그를 시간 축으로 누적한 패널 데이터로 구축 (Phase 2 고도화
  입력).
- 모든 화면·API 응답에 **공공데이터 출처 명시**(공고 요건 준수).

## 5. 개발 결과물의 기술성과 구현 및 실현가능성

### 5.1 활용한 주요 기술

| 레이어 | 기술 | 선정 사유 |
|---|---|---|
| 프레임워크 | Next.js 16 App Router + TypeScript | SSR, Vercel 즉시 배포 |
| 스타일 | Tailwind CSS v4 | 빠른 프로토타이핑 |
| 지도 | Leaflet + OpenStreetMap | 무료, API 키 불필요 |
| 상태 관리 | TanStack Query + Zustand persist | 자동 리페치, 체크인 영속 |
| 차트 | Recharts | 혼잡 예측 막대 그래프 |
| AI (로컬) | **Ollama `llama3.3:70b-q4`** + `outlines` 구조화 출력 | 온디바이스 Tool-use, 서버 전송 없음, 오프라인 시연 가능 |
| DB (Phase 2) | Supabase (PostgreSQL) | 체크인 로그·인증·RLS |
| 배포 | Vercel | 무료 티어, GitHub 연동 |

### 5.2 구현 상태 (MVP 마일스톤)

- 5개 MVP 화면 와이어프레임 확정
- 공공데이터 5종 프록시 + AI 2종 엔드포인트 구조 확정
- 서울·수원·부산 일부 POI **시연 셋** 구축

**제출물**: Vercel 배포 URL + 접속 QR + 화면 녹화 영상.

### 5.3 실현 가능성 평가

| 리스크 | 영향도 | 대응 |
|---|:---:|---|
| 공공 API 일시 장애 | 높음 | 5분 TTL 캐시 + stale-while-revalidate + mock 폴백 |
| 지자체별 속성 스키마 상이 | 중간 | 표준화 어댑터 레이어 |
| 수유실 데이터 최신성 | 중간 | 사용자 제보 + 지자체 연 1회 일괄 갱신 |
| 개인정보 (위치) | 높음 | 클라이언트에서만 사용, PII 서버 저장 최소화 |

## 6. 기대 효과

### 6.1 사회적 가치 창출

- **가족 여가 기회 회복 (직접 효과)**
  부모의 외출 의사결정 30분 한계 [7]를 **10분 이내로 단축**하여 "정보 단절로
  포기되는 외출"을 연간 **1,000만 회 이상** 회복 가능. 영유아 양육가구 약 180만
  가구 [16] × 월 1회 추가 외출 가정.

- **공공 편의시설 활용률 제고**
  전국 공중화장실 8만 5천여 개소 [3]·수유실 3천여 개소 [4]·어린이도서관 1천여
  개관 [6]의 유휴 정보 접근성을 단일 채널로 통합하여 **인지율 2배** 이상 향상.

- **저출생 체감도 완화**
  정부 저출생 대응 예산 [17]이 "결혼·출산 지원금"에 집중되는 가운데, 본 서비스는
  **양육 단계의 일상 체감 지원**을 디지털 공공재로 보강한다.

- **장애인 가족 포용**
  장애인화장실·엘리베이터·경사로를 묶은 **장애인 가족 모드**는 "교통약자 이동지원"
  정책 흐름 [5]과 직접 정합하며, 기존 육아 앱에 없는 포용성을 제공한다.

### 6.2 상생협력·일자리 창출 효과

- **지자체·관광재단 확산**: 지역 축제·박물관 정보와 결합한 지자체 커스터마이징
  용역 수요 창출.
- **육아 크리에이터 B2C**: 부모 사용자가 공유한 "오늘의 코스"를 콘텐츠화하여
  신규 수익 모델 형성.
- **아이돌봄 플랫폼 제휴**: 정부 아이돌봄서비스(여성가족부)와 동선 연계 [2].

### 6.3 정량 목표 (론칭 후 6~12개월)

| 지표 | 6개월 목표 | 12개월 목표 |
|---|---|---|
| 월 활성 이용자 (MAU) | 5만 | 20만 |
| "오늘 나들이" 카드 생성 | 30만 건 | 200만 건 |
| 체크인 수 | 20만 | 150만 |
| 재사용률 (주간) | 30% | 50% |
| 연계 POI 수 | 수도권 30,000 | 전국 200,000 |

### 6.4 공공가치 선순환

> **체크인 축적 → 혼잡 예측 정확도 상승 → AI 동선 만족도 상승
> → 이용자 확산 → 더 풍부한 데이터**

이 선순환은 저출생 시대 **양육가구 디지털 지원의 모범 사례**로 자리잡는다.
