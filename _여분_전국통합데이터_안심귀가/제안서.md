## 1. 개발 결과물 개요

**안심귀가**는 **지자체 안심귀가 스카우트·공공 CCTV·가로등·24시 편의점·지구대
좌표**를 결합해, 단순 최단 경로가 아닌 **"가장 밝고 사람이 있는 길"** 을 실시간
경로로 그려주는 반응형 안심 내비게이션이다.

경찰청 「범죄통계」 기준 한국의 강력·절도·폭력 등 주요 범죄의 상당수는 **심야 시간대
(22시~06시)** 에 집중되며 [1], 여성가족부·서울시 등은 **여성안심귀가 스카우트
서비스**를 전국적으로 확산해 2024년 기준 **연간 수십만 건**이 이용되고 있다 [2].
그러나 시민이 **"오늘 밤, 지금 내가 걷는 길이 안전한가"** 를 실시간으로 판단할
통합 도구는 부재하다. 안심귀가는 **공공 CCTV·가로등·편의점·지구대** 를 보안 그래프의
노드로 통합하고, 각 경로의 **밝기·인적 밀도·감시 커버리지**를 점수화해 경로
가중치로 삼는다.

**한 줄 요약**: "공공데이터가 그려주는 가장 안전한 귀가 경로. 출발 버튼 하나로
보호자에게 실시간 위치가 흐르고, 위험 징후가 감지되면 112까지 자동 연결."

## 2. 개발 결과물의 기능 및 특징

### 2.1 서비스의 목적 및 배경

#### 문제 정의

- **심야 범죄와 고립 보행자 안전은 여전히 핵심 치안 과제다.**
  경찰청 「범죄통계」에 따르면 주요 강력·절도 범죄의 유의미한 비중이 심야 시간대에
  발생하고 있으며, 여성·청소년 대상 범죄에서 귀가 경로 안전성은 여전히 중대 이슈다
  [1]. 경찰청·국가통계포털(KOSIS)은 범죄 통계를 연례 공표하고 있다 [3].

- **안심귀가 스카우트 서비스는 존재하지만 예약 피로가 크다.**
  여성가족부·서울시 여성안심귀가 스카우트는 사전 신청·당일 신청의 이중 채널로
  운영되며 [2], 실제 이용자는 "신청이 번거롭다", "배정까지 대기가 길다" 등의
  진입 장벽을 호소한다. 공공 서비스가 **지도·경로와 결합**되지 않으면 이용률이
  정체된다.

- **공공 CCTV·가로등은 많지만 '어디에' 있는지 시민은 모른다.**
  행정안전부·지자체는 전국 약 **130만 대 이상의 공공 CCTV**를 운영하는 것으로
  발표되어 왔으며 [4], 지자체별 가로등·보안등 데이터셋이 공공데이터포털에
  개방되어 있다 [5]. 그러나 시민 관점에서 **경로상 어떤 길에 CCTV가 있고 가로등이
  몇 m 간격인지** 를 보여주는 앱은 없다.

- **24시 편의점·지구대는 '심야 피난처' 역할을 한다.**
  편의점업계 자료에 따르면 전국 편의점은 **5만 개소를 넘어**서며 심야 운영 비율이
  높다 [6]. 경찰청 지구대·파출소 역시 공공데이터로 좌표가 공개되어 있어 [7],
  이 두 축을 **"가까운 안전지점"** 으로 묶어 경로 결정에 반영할 수 있다.

- **개인정보·편집증 없는 안심 공유 수단이 부재하다.**
  가족·친구에게 위치를 실시간으로 공유할 때, 상대가 **별도 앱을 설치하는 부담**
  없이 간단한 SMS 링크 하나로 확인할 수 있는 공공·비영리 수단은 드물다. 민간
  위치공유 앱은 상시 위치 추적·광고·과금 부담이 있다.

#### 해결 방향

1. **안심귀가 스카우트·CCTV·가로등·편의점·지구대**를 보안 그래프의 노드로 통합.
2. **"밝기 점수"** 가중치 경로 탐색: 짧은 길이 아닌 **가장 안전한 길**을 우선.
3. **보호자 실시간 공유 링크** (SMS 웹 링크, 설치 불필요).
4. **긴급 흔들기** → 앱이 위험 상황으로 분류 시 112 + 보호자 동시 통보.

### 2.2 서비스의 핵심 기술·기능·구성·특징

#### (1) 5대 핵심 화면

| # | 화면 | 핵심 기능 |
|:--:|---|---|
| ① | **홈 (오늘의 귀가)** | 도착지 입력 → 🟢 밝은 경로 / 🟡 보통 / 🔴 어두움 3개 후보 + 예상 소요시간 |
| ② | **경로 지도** | CCTV·가로등·편의점·지구대 마커 + 안심귀가 스카우트 신청 버튼 |
| ③ | **보호자 공유** | SMS 링크 1회 발송, 도착 시 자동 종료, 3분 단위 위치 스냅샷 |
| ④ | **긴급 모드** | 흔들기 감지 → 음성 안내 3초 카운트 → 112 + 보호자 동시 알림 |
| ⑤ | **리포트** | 최근 귀가 평균 '밝기 점수' + 가장 많이 이용한 공공 인프라 |

#### (2) AI 기능 (공모전 가점 10점 대상)

- **"밝기 점수" 경로 가중치 모델 `/api/ai/safepath`**
  각 그래프 엣지에 대해: CCTV 커버리지(반경 50m)·가로등 밀도·편의점/지구대
  도보 거리·유동인구 대용 지표(이동통신 공공통계 [8])·시간대 보정치를 가중 합산해
  **밝기 점수 S ∈ [0, 1]** 을 계산한다. A* 경로 탐색 시 **거리 + (1-S)·α** 를 비용
  함수로 사용해 "밝은 길"이 우선되도록 한다.

- **긴급 상황 자연어 분류 `/api/ai/triage`**
  사용자가 음성/텍스트로 "누가 따라와요" 같은 입력을 주면 LLM Tool-use가
  **"일상 불안 / 잠재 위협 / 즉시 긴급"** 3단계로 분류하고 각 단계별 권고
  (가까운 편의점 안내 / 보호자 긴급 공유 / 112 연결)를 제시한다 [9].

- **주간 안전 요약** *(Phase 2)*
  LLM이 "이번 주 평균 밝기 점수 0.78, 가장 많이 이용한 편의점은 ○○점"과 같은
  요약을 자동 생성.

#### (3) 시스템 구성

```
[Frontend: Next.js 16 App Router · TypeScript · Tailwind CSS v4]
  └─ 긴급 UI: 화면 단순화, 밝은 대비, 음성 안내
          ↓
[Backend API Routes — 공공데이터 프록시 + 5분 TTL 캐시]
  ├─ /api/scouts?near=lat,lng              (안심귀가 스카우트)
  ├─ /api/cctv?near=lat,lng                (공공 CCTV)
  ├─ /api/lamps?near=lat,lng               (가로등·보안등)
  ├─ /api/cvs?near=lat,lng&open24=1        (24시 편의점)
  ├─ /api/stations?near=lat,lng            (지구대·파출소)
  ├─ /api/ai/safepath                      (밝기 경로)
  └─ /api/ai/triage                        (긴급 분류)
          ↓
[공공데이터포털 API] [LLM API] [SMS Gateway] [112 연계 (자치경찰 API)]
```

- **경로 엔진**: OSRM + 밝기 가중치 후처리.
- **긴급 프로토콜**: 흔들기 감지 → 진동·음성 3초 카운트다운 (오탐 방지) → 112 +
  보호자 SMS 동시 발송.
- **PII 최소화**: 보호자 공유 토큰은 서버에 저장하지 않고 JWT 만료형 URL.

## 3. 기존 서비스와 차별성 및 독창성

### 3.1 기존 유사 서비스 비교

| 항목 | 지자체 안심귀가 앱 | 민간 지도앱 | 민간 위치공유 앱 | **안심귀가** |
|---|:---:|:---:|:---:|:---:|
| 밝기 점수 경로 | ❌ | ❌ | ❌ | **✅ (전국)** |
| 안심귀가 스카우트 지도 연동 | ⚠️ | ❌ | ❌ | **✅** |
| CCTV·가로등 통합 | ❌ | ❌ | ❌ | **✅** |
| 24시 편의점·지구대 필터 | ❌ | 부분 | ❌ | **✅ (안전지점)** |
| 보호자 공유 (앱 설치 불요) | ❌ | ❌ | ❌ | **✅** |
| 긴급 흔들기 → 112 | ⚠️ (일부) | ❌ | ❌ | **✅** |
| 이용료 | 무료 | 무료 | 부분 유료 | **무료** |

### 3.2 독창적·창의적 요소

1. **밝기 점수 경로 탐색**
   짧은 길이 아니라 **CCTV·가로등·편의점·지구대 밀도**로 환산한 "밝기 점수"를
   경로 가중치로 쓰는 접근은 공공 영역에 부재하다. 안전공학·도시계획에서
   "환경 설계 범죄예방(CPTED)" 이론이 강조하는 **자연적 감시(Natural Surveillance)**
   를 공공데이터 레이어로 구현한다 [10].

2. **안심귀가 스카우트의 UX 통합**
   기존 서비스의 별도 앱·홈페이지 신청 플로우를 **경로 추천 → 예상 위험 구간
   사전 신청**으로 묶어 "이 구간부터 동행 요청" 1탭 플로우로 단순화한다.

3. **"앱 설치 없는 보호자 공유"**
   가족에게 앱 설치를 요구하지 않는다. SMS 웹 링크 하나로 실시간 위치·도착 알림을
   전달. 설치 장벽을 0으로 만들어 사용률을 극대화한다.

4. **긴급 자연어 분류 에이전트**
   흔들기·버튼 기반 SOS와 달리 **음성/텍스트 맥락을 분류**해 3단계 대응 결정.
   오탐으로 인한 112 낭비를 줄이고, 실제 긴급 상황에서는 **3초 카운트다운** 내에
   자동 연결. 긴급 서비스 부담 감소 + 사용자 보호를 동시에 달성.

### 3.3 지식재산권 전략

- "공공 감시·편의 인프라 밀도 기반 안심 경로 탐색 시스템 및 방법" BM 특허 출원.
- 경로 가중치 계산기 MIT 라이선스 공개.

## 4. 공공데이터 등 데이터 활용성

### 4.1 필수 활용 공공데이터

| 데이터명 | 원천 기관 | 획득 방법 | 서비스 내 활용 |
|---|---|---|---|
| **여성안심귀가 스카우트** | 여성가족부·지자체 [2] | 공공데이터포털 + 지자체 사이트 | 경로상 동행 서비스 신청 |
| **공공 CCTV 위치** | 행정안전부·지자체 [4] | 공공데이터포털 | 경로 가중치, 마커 |
| **가로등·보안등 위치** | 지자체 [5] | 공공데이터포털 CSV | 경로 가중치 |
| **편의점 주소·운영시간** | 공정거래위·각 브랜드 공개 [6] | 공공데이터포털 + 민관 연계 | 24시 필터, 피난처 |
| **지구대·파출소 좌표** | 경찰청 [7] | 공공데이터포털 | 안전지점 마커 |
| **유동인구 이동통신 통계** | 행정안전부·SKT [8] | 공공데이터포털 | 시간대 가중치 |

### 4.2 타 기관 공공·민간 데이터 보조 활용

| 데이터 | 출처 | 활용 목적 |
|---|---|---|
| 지구 조도·일몰 시간 | 기상청 | 야간 모드 자동 전환 |
| 지도·경로 | OpenStreetMap + OSRM | 도보 경로 탐색 |

### 4.3 공공데이터 획득의 지속성

- 활용 데이터 다수가 공공데이터포털 상시 개방 항목이며 일부는 국가중점데이터로
  지속성이 보장된다 [11].
- CCTV·가로등 데이터는 갱신 주기가 분기 단위로 길다. **월 1회 정기 동기화 + 사용자
  제보형 보정** 을 병행.
- 112 연계는 **자치경찰 API 협약** 또는 `tel:112` 딥링크 폴백으로 구현.

### 4.4 데이터 가공 및 결합

- 공공 인프라는 위경도로 색인화되어 반경 50m 커널로 **밀도 지도(Density Grid)** 를
  구성.
- 그래프 엣지에 대해 밀도·시간대·유동인구를 가중 합산 → 밝기 점수 S 계산.
- 모든 화면·API 응답에 **공공데이터 출처 명시** (공고 요건 준수).

## 5. 개발 결과물의 기술성과 구현 및 실현가능성

### 5.1 활용한 주요 기술

| 레이어 | 기술 | 선정 사유 |
|---|---|---|
| 프레임워크 | Next.js 16 App Router + TypeScript | SSR, Vercel 즉시 배포 |
| 스타일 | Tailwind CSS v4 | 긴급 UI 고대비 |
| 지도·경로 | Leaflet + OSRM (self-host) | 가중치 커스터마이징 |
| 상태 관리 | Zustand persist | 최근 경로·보호자 목록 |
| AI | Anthropic Claude Haiku 4.5 (Tool-use) | 저비용, 구조화된 함수 호출 |
| 긴급 | Geolocation + DeviceMotion API | 흔들기 감지 |
| SMS | NHN Toast SMS Gateway | 보호자 링크 발송 |
| 배포 | Vercel | 무료 티어, 자동배포 |

### 5.2 구현 상태 (MVP 마일스톤)

- 5개 MVP 화면 와이어프레임 확정
- 공공데이터 6종 프록시 + AI 2종 엔드포인트 구조 확정
- 서울 관악·동작 일부 구간 **밝기 점수 시뮬레이션 셋** 준비

**제출물**: Vercel 배포 URL + 접속 QR + 시연 영상.

### 5.3 실현 가능성 평가

| 리스크 | 영향도 | 대응 |
|---|:---:|---|
| 112 직접 연계 미승인 | 높음 | `tel:112` 딥링크 + 문자 메시지 폴백 |
| CCTV·가로등 스키마 상이 | 중간 | 표준화 어댑터 |
| 오탐 긴급 발동 | 높음 | 3초 카운트다운 + 해제 버튼 |
| 개인정보 (경로) | 치명 | 보호자 토큰 JWT 만료형, 서버 저장 최소 |
| 심야 API 트래픽 피크 | 중간 | 5분 TTL + CDN |

### 5.4 향후 고도화 로드맵

| Phase | 기간 | 핵심 작업 |
|:--:|---|---|
| **Phase 1** | 출시 후 3개월 | 사용자 제보(어둡다/밝다) 반영, 가중치 학습 |
| **Phase 2** | 4~6개월 | 지자체 자치경찰 API 협약, 112 연계 정식화 |
| **Phase 3** | 7~12개월 | 학교·기숙사 안심귀가 B2G, 고령자·청소년 모드 |
| **Phase 4** | 12개월 이후 | 기상·재난경보(폭설·지진) 안심 동선 확장 |

## 6. 기대 효과

### 6.1 사회적 가치 창출

- **심야 범죄 피해 감소**
  CPTED 이론 [10]에 따르면 **자연적 감시**가 강화된 경로는 범죄 기회 자체를 축소.
  본 서비스가 이용자를 고감시 경로로 유도하면 **심야 범죄 취약 구간 우회** 효과가
  기대된다. 경찰청 범죄통계를 지속적으로 피드백하여 효과를 정량 측정 [1][3].

- **여성안심귀가 스카우트 이용 활성화**
  여성가족부·지자체 스카우트 서비스 [2]의 신청 플로우를 앱 내로 흡수해 이용률을
  2배 이상 끌어올리는 것을 목표로 한다.

- **112·경찰 인력 운영 효율**
  긴급 자연어 분류로 **허위·과잉 신고를 감소**시키고, 진짜 긴급 상황만 112로
  에스컬레이션. 경찰 인력의 효율적 운영에 기여.

- **디지털 성평등**
  여성·청소년·야간 근무자의 **도시 이동 자유도**를 공공데이터로 확장. 포용적
  도시 설계의 디지털 연장.

### 6.2 상생협력·일자리 창출 효과

- **지자체 B2G**: 지역 맞춤 밝기 점수·안심귀가 스카우트 연계 모듈 유상 제공.
- **대학·기숙사 B2B**: 안전 귀가 대시보드.
- **시민 참여**: 가로등 고장·어두운 길 제보 → 지자체 시설 개선 파이프라인.

### 6.3 정량 목표 (론칭 후 6~12개월)

| 지표 | 6개월 목표 | 12개월 목표 |
|---|---|---|
| 월 활성 이용자 (MAU) | 5만 | 30만 |
| 안심 귀가 완료 세션 | 20만 | 300만 |
| 보호자 공유 링크 활성 | 3만 | 50만 |
| 안심귀가 스카우트 신청 유도 | 1천 | 5만 |
| 사용자 제보 (어두움/고장) | 5천 | 10만 |

### 6.4 공공가치 선순환

> **제보·경로 데이터 축적 → 밝기 점수 고도화 →
> 지자체 인프라 정비 데이터 → 도시 안전 향상**

공공데이터의 민·관 환류 모범 사례.
