## 1. 개발 결과물 개요

**실버친구**는 65세 이상 고령 이용자를 위해 **경로당·무더위쉼터·공원·보건소·저상버스
실시간 정보**를 단일 지도에 통합하고, 글씨·음성·고대비 UX를 디폴트로 채택한
**고령자 전용 공공 생활 지원 반응형 웹앱**이다. 통계청 장래인구추계 기준 2025년
한국의 65세 이상 인구는 **1,051만 명**(전체의 20.3%)을 돌파해 초고령사회에 진입했고
[1], 동시에 고령층의 디지털 접근성은 세대 평균의 절반 수준에 머물러 있어 [2]
공공자원의 존재가 곧 이용으로 이어지지 않는 구조적 단절이 심화되고 있다.

이용자는 화면 하단 큰 버튼 한 번으로 **"오늘 갈 곳" 추천 카드**를 받아 가까운
경로당 또는 무더위쉼터까지의 저상버스 도착시간을 확인하고, 보호자에게 출발·도착을
**자동 문자**로 공유한다. 음성으로 "오늘 덥다, 에어컨 있는 데 없어?"라고 물으면
LLM 에이전트가 실시간 무더위쉼터·경로당 에어컨 가동 여부 API를 Tool-use로 조회해
문장형으로 답한다. 이 전 과정이 **단일 앱 한 플로우**에 묶인다는 점이 각 지자체
홈페이지·개별 앱에 흩어진 기존 서비스와의 결정적 차별점이다.

**한 줄 요약**: "경로당 가는 길·버스·쉼터·자식 안심문자까지 한 화면. 어르신을 위한
공공데이터 통합 나들이 앱."

## 2. 개발 결과물의 기능 및 특징

### 2.1 서비스의 목적 및 배경

#### 문제 정의

- **초고령사회 진입, 고립 고령자 급증.**
  통계청은 2024년 12월 한국이 65세 이상 인구 비율 20%를 돌파해 **초고령사회**에
  진입했다고 공식 발표했다 [1]. 65세 이상 1인 가구는 2023년 기준 **213만 8천 가구**로
  고령 가구의 **37.8%**에 이르며 [3], 이들 상당수는 지역사회 자원(경로당·복지관·
  무더위쉼터)으로 연결되는 **디지털 가교를 갖지 못한** 채 고립되어 있다.

- **고령자의 디지털 정보격차는 여전히 뚜렷하다.**
  과학기술정보통신부·한국지능정보사회진흥원 「2024 디지털정보격차 실태조사」에
  따르면, 고령층의 **디지털정보화 종합 수준은 일반국민 대비 71.8%**이며 특히
  **활용 수준은 64.6%**에 그쳐 접근성·애플리케이션 활용 장벽이 크다 [2]. 고령자가
  "앱을 다운받아 회원가입하고 주소를 검색"하는 기존 서비스 플로우 자체가 진입
  장벽이다.

- **기후재난 시 고령자 사망은 반복된다.**
  질병관리청 온열질환 응급실감시체계에 따르면 2024년 온열질환자 중 **65세 이상이
  31.9%**, 온열질환 추정 사망자의 **80% 이상이 고령자**였다 [4]. 행정안전부는 전국
  **약 5만여 개의 무더위쉼터**를 지정·운영하고 있으나 [5], 어르신이 **가장 가까운
  쉼터를 실시간으로 찾는 수단**은 파편화되어 있다.

- **경로당은 있는데 비어 있고, 필요한 사람은 찾지 못한다.**
  보건복지부·한국노인종합복지관협회 자료 기준 전국 **경로당은 6만 7,000여 개소**로
  노인복지시설 중 압도적 다수를 차지하지만 [6], 등록 회원 중심으로 운영되는 경우가
  많아 신규 이사·방문자가 접근하기 어렵고, 시설별 **에어컨·냉난방·프로그램 정보가
  일원화된 디지털 창구**가 없다.

- **저상버스는 늘었지만 "언제 오는지" 모르면 의미가 없다.**
  국토교통부는 저상버스 도입률을 지속 확대하고 있으며 도시형 시내버스 기준
  2024년 도입률이 **40%대**에 도달했다 [7]. 그러나 고령자의 버스 이용 포기 1순위는
  "언제 오는지 모른다"다. 실시간 정보가 **큰 글씨·음성**으로 제공되지 않으면
  저상버스의 배리어프리 가치는 절반에 그친다.

#### 해결 방향

1. **공공데이터포털 통합 API(경로당·쉼터·저상버스·공원·보건소)를 단일 지도로 병합**.
2. **"큰 글씨 · 고대비 · 음성 대화"를 디폴트**로 채택해 고령자 디지털 격차를 완화.
3. **"오늘 갈 곳" AI 추천**이 날씨·미세먼지·요일·거리를 결합해 하루 한 장의 카드로
   요약.
4. **보호자 공유 링크**(별도 앱 설치 불필요)로 자녀 세대가 출발·도착을 실시간 확인.

### 2.2 서비스의 핵심 기술·기능·구성·특징

#### (1) 5대 핵심 화면

| # | 화면 | 핵심 기능 |
|:--:|---|---|
| ① | **홈 (오늘 카드)** | 내 위치 기반 "오늘 어디 갈까요?" 카드 1장: 추천 경로당 / 쉼터 / 공원 1곳 + 저상버스 도착시간 + 날씨·미세먼지 경보 |
| ② | **가까운 쉼터·경로당** | 반경 1km 마커 지도 + 리스트. 에어컨 🟢 / 난방 🟠 / 운영중 여부 색상 코딩. 도보·버스 경로 자동 비교 |
| ③ | **음성 비서** | "오늘 덥다, 에어컨 있는 데 어디야?" 등 자연어 질의. LLM Tool-use가 쉼터·보건소·공원 API를 실시간 조회 [8] |
| ④ | **버스 알림** | 저상버스 실시간 위치·도착예정시간, 정류장 번호 큰 숫자 표시, 3분 전 음성 알림 |
| ⑤ | **보호자 공유** | 가족 휴대폰번호만 등록하면 SMS 웹 링크로 실시간 위치·도착 알림 전송 (별도 앱 설치 불필요) |

#### (2) AI 기능 (공모전 가점 10점 대상)

- **"오늘 갈 곳" 추천 랭킹 `/api/ai/today`**
  입력: 사용자 위치·연령·시간대·기상청 예보·대기질·요일. 처리: 거리 가중치 0.4,
  온열·한파 위험도 0.3, 혼잡 예측 0.2, 선호 이력 0.1의 선형 랭킹 모델로 시작해
  체크인 로그가 쌓이면 LightGBM·LambdaMART로 고도화한다. 고령자 이동·외출 빈도
  연구에 따르면 "**하루 1회 외출**"이 인지건강·사회참여에 유의미한 효과를 보이는
  핵심 지표로 제시되어 [9], 본 추천은 이 지표를 목표 변수로 삼는다.

- **자연어 음성 비서 `/api/ai/chat`**
  LLM Tool-use 패턴 [8]. LLM은 다음 도구를 필요 시 호출한다:
  `nearby_shelters(lat, lng, type)`, `bus_arrival(stop_id)`,
  `nearby_parks(lat, lng)`, `nearby_clinics(lat, lng)`, `weather_now(lat, lng)`.
  "비 와서 나가기 싫은데 가까운 경로당" 같은 복합 조건을 실사실 기반으로 답한다.

- **보호자용 주간 리포트** *(Phase 2)*
  LLM이 "이번 주 외출 3회, 평균 산책 28분 — 지난 주 대비 외출 1회 증가"와 같은
  가족 공유용 요약을 자동 생성한다.

#### (3) 시스템 구성

```
[Frontend: Next.js 16 App Router · TypeScript · Tailwind CSS v4]
  └─ 고령자 UI 토큰: 최소 18px, 대비 7:1 이상, 터치영역 48×48dp 이상 (WCAG 2.2 AAA)
          ↓
[Backend API Routes — 공공데이터 프록시 + 5분 TTL 캐시]
  ├─ /api/shelters?near=lat,lng            (무더위/한파 쉼터)
  ├─ /api/seniorhalls?near=lat,lng         (경로당)
  ├─ /api/parks?near=lat,lng               (공원·산책로)
  ├─ /api/bus/arrival?stopId=...           (저상버스 실시간)
  ├─ /api/clinics?near=lat,lng             (보건소·방문간호)
  ├─ /api/ai/today                         (오늘 추천)
  └─ /api/ai/chat                          (음성 비서)
          ↓
[공공데이터포털 API] [LLM API] [Kakao TTS/STT] [카카오 알림톡/문자 Gateway]
```

- **공공 API 프록시 레이어**: 서버에 인증키를 보관하고 5분 TTL + stale-while-revalidate
  캐시. 상류 API 단기 장애 시 직전 캐시로 응답.
- **지도**: Leaflet + OpenStreetMap(무료). 마커는 컬러블라인드 친화 팔레트와
  픽토그램 이중 인코딩(색 + 아이콘)으로 고대비를 달성.
- **음성**: 브라우저 Web Speech API 1차, 고령자 발화 인식률 이슈 발생 시
  **whisper.cpp(로컬 large-v3)** 로 폴백. 서버 전송 없이 온디바이스 처리.

## 3. 기존 서비스와 차별성 및 독창성

### 3.1 기존 유사 서비스 비교

| 항목 | 각 지자체 복지포털 | 민간 지도앱 | 경로당/복지관 홈페이지 | **실버친구** |
|---|:---:|:---:|:---:|:---:|
| 경로당 실시간 운영 정보 | ⚠️ (일부) | ❌ | ⚠️ (개별) | **✅ (전국 통합)** |

| 저상버스 도착시간 | ❌ | ✅ (일반 버스와 혼재) | ❌ | **✅ (저상만 필터)** |

### 3.2 독창적·창의적 요소

1. **"고령자 디폴트" UX 토큰 시스템**
   기존 앱은 "접근성 모드"를 옵션으로 제공하지만 실버친구는 **고령자 토큰을
   디폴트**로 채택한다. WCAG 2.2 AAA 수준의 대비·터치영역·글자크기를 기본값으로
   설정하고, 40대 이하 보호자에게만 "작은 글씨로 보기" 옵션을 제공한다 [10].
   이는 "Accessibility is not an option, it is the default"라는 Microsoft Inclusive
   Design 원칙과 일치한다 [11].

2. **공공자원 체인 라우팅 (Chained Public-Resource Routing)**
   "경로당 + 저상버스 + 보건소"를 시공간 체인으로 묶어 **한 외출 = 한 플로우**로
   설계한다. 국가 MaaS(K-MaaS) 전략이 교통수단 간 연계에 주력하는 것과 달리 [12],
   본 서비스는 **복지 자원까지 MaaS의 대상**으로 확장한다.

3. **보호자 "앱 없는 공유" 링크**
   자녀에게 "또 앱 설치하세요"를 요구하지 않는다. SMS 웹 링크 하나로 위치·도착을
   실시간 확인하게 하여 **설치 장벽을 0으로** 만든다. 행정안전부 공공알림문자
   가이드의 웹 기반 메시지 체계와 호환된다 [13].

4. **외출 리워드 피드백 루프**
   하루 한 번 외출 체크인이 축적되면 "이번 주 외출 3회" 카드와 함께 보호자에게
   공유된다. 노인 외출 빈도가 우울감·인지저하와 역상관을 보인다는 한국보건사회연구원
   보고 [9]를 UX 설계의 1차 지표로 삼는다.

### 3.3 지식재산권 전략

- "고령자 디폴트 UX 토큰 시스템 및 공공자원 체인 추천 방법" 비즈니스모델 특허 출원
  검토.
- 핵심 API 프록시·캐시 레이어는 MIT 라이선스 오픈소스로 공개하여 지자체·복지관
  재활용을 유도한다.

## 4. 공공데이터 등 데이터 활용성

### 4.1 필수 활용 공공데이터

모두 **공공데이터포털(data.go.kr)** 에서 오픈 API로 제공되며, 상당수는 2024~2025년
행정안전부·한국지역정보개발원의 전국 통합개방 대상에 포함되었다 [14].

| 데이터명 | 원천 기관 | 획득 방법 | 서비스 내 활용 |
|---|---|---|---|
| **경로당 현황 정보** | 보건복지부·지자체 | 공공데이터포털 REST API, 주소·좌표·운영시간 | 홈 카드, 지도 마커 |
| **무더위쉼터·한파쉼터 위치** | 행정안전부 | 국민재난안전포털 / 공공데이터포털 | 날씨 경보 시 최우선 추천 |
| **저상버스 실시간 정보** | 국토교통부·지자체 | TAGO(국가대중교통정보센터) / 공공데이터포털 | 버스 도착시간 |
| **도시공원 정보** | 국토교통부·한국토지주택공사 | 공공데이터포털 CSV/API | 산책 루트 |
| **보건소·보건지소** | 보건복지부 | 공공데이터포털 | 방문간호·예방접종 안내 |

### 4.2 타 기관 공공·민간 데이터 보조 활용

| 데이터 | 출처 | 활용 목적 |
|---|---|---|
| 단기 기상·온열질환 예보 | 기상청 API허브 [15] | 폭염·한파 경보 시 강제 쉼터 추천 |
| 대기질 예보 (미세먼지) | 에어코리아 [16] | 외출 권고 레벨 산정 |
| 지도 타일 | OpenStreetMap | 지도 시각화 (API 키 불필요) |

### 4.3 공공데이터 획득의 지속성

- 활용 API는 공공데이터포털의 상시 개방 데이터이며, 통합개방 데이터는
  **국가중점데이터**로 지속성이 제도적으로 보장된다. 2024년 말 기준 국가중점데이터는
  217종, 공공데이터 개방 건수는 10만 건을 상회한다 [17].
- 인증키 만료·정기 순단 대비 **5분 TTL 인메모리 캐시 + stale-while-revalidate**
  전략으로 상류 API 단기 장애에 강건.
- 데이터 갱신 주기(경로당 일 1회, 쉼터 시간당, 버스 30초)를 각각 존중해 Rate Limit
  초과를 방지한다.

### 4.4 데이터 가공 및 결합

- 모든 데이터는 **위경도(Geo) 공통 키**로 결합. Haversine 거리로 반경 1km 내
  자원만 필터.
- 날씨·대기질은 실시간 경보 상태에 따라 추천 가중치가 동적으로 재계산된다. 예:
  폭염특보 → 무더위쉼터 가중치 0.3 → 0.7.
- 화면·API 응답에 **공공데이터 출처 명시**(공고 요건 준수).

## 5. 개발 결과물의 기술성과 구현 및 실현가능성

### 5.1 활용한 주요 기술

| 레이어 | 기술 | 선정 사유 |
|---|---|---|
| 프레임워크 | Next.js 16 App Router + TypeScript | SSR 초기 로딩, Vercel 즉시 배포 |
| 스타일 | Tailwind CSS v4 + 고령자 디자인 토큰 | 최소 18px·7:1 대비 디폴트 보장 |
| 지도 | Leaflet + OpenStreetMap | 무료, API 키 불필요 |
| 상태 관리 | TanStack Query + Zustand persist | 자동 리페치, 비회원 외출 로그 영속 |
| 음성 (STT) | Web Speech API (1차) / **whisper.cpp · MLX Whisper** (로컬 폴백) | 고령자 발화 인식률 확보, 서버 전송 없음 |
| TTS | **Kokoro TTS (로컬, 한국어)** | 자연스러운 한국어 합성, 오프라인 동작 |
| AI (로컬) | **Ollama `llama3.3:70b-q4`** + `outlines` 구조화 출력 | 로컬 Tool-use, 네트워크 불필요, 개인정보 보호 |
| 알림 | SMS Gateway (NHN Toast) | 보호자용, 앱 설치 불요 |
| 배포 | Vercel | 무료 티어, GitHub 연동 자동배포 |

### 5.2 구현 상태 (MVP 마일스톤)

- 5개 MVP 화면 와이어프레임 확정 (홈 · 주변 쉼터 · 음성비서 · 버스 · 보호자)
- 공공데이터 5종 프록시 스켈레톤 + AI 2종 엔드포인트 구조 확정
- 큰 글씨·고대비 디자인 토큰 기본값 정의 (최소 18px / 대비 7:1)
- 서울·수원·성남 일부 경로당·쉼터 좌표 **시연 셋** 준비

**제출물**: Vercel 배포 URL + 접속 QR + 화면 녹화 영상.

### 5.3 실현 가능성 평가

| 리스크 | 영향도 | 대응 |
|---|:---:|---|
| 고령자 발화 인식 오류 | 높음 | Web Speech 1차 실패 시 **whisper.cpp(로컬)** 자동 폴백, 짧은 명령어 UX |
| 공공 API 일시 장애 | 높음 | 5분 TTL 캐시 + stale-while-revalidate + mock 폴백 |
| 지자체별 스키마 상이 | 중간 | 표준화 어댑터 레이어 + 지역 우선순위 |
| 개인정보 (보호자 번호) | 높음 | 해시 저장, 번호는 클라이언트에만 보관, PII 서버 최소화 |
| 인증키 발급 지연 | 중간 | 데모 모드 내장 (시연 셋) |

## 6. 기대 효과

### 6.1 사회적 가치 창출

- **고독 완화 (직접 효과)**
  65세 이상 1인 가구 213만 명 [3] 중 본 서비스로 월 1회 이상 외출이 늘어나는
  이용자가 **10%(약 21만 명)** 에 도달한다면, 「외출 빈도–우울감 역상관」 연구 [9]
  를 바탕으로 우울 위험군 감소 효과가 기대된다.

- **온열질환·한파 사망 예방 (생명 가치)**
  2024년 온열질환자의 31.9%가 65세 이상 [4]. 전국 5만여 무더위쉼터 [5]를 **실시간
  거리·운영 여부**로 탐색 가능하게 하여 "쉼터가 있어도 못 가는" 구조적 단절을
  해소한다.

- **공공자원 활용률 제고**
  경로당 6만 7천여 개소 [6]의 유휴 활용률을 높이고, 신규 이사·방문자의 진입 장벽을
  낮춰 **지역사회 통합돌봄**의 디지털 출발점이 된다.

- **저상버스의 실질 가치 구현**
  저상버스 도입률 40%대 [7]에도 불구하고 "언제 오는지 모른다"는 정보 격차로 이용
  포기가 빈번했다. 실시간 도착시간 음성 알림이 저상버스의 배리어프리 가치를 실제
  이동으로 전환한다.

### 6.2 상생협력·일자리 창출 효과

- **지자체 확산 모델**: 핵심 로직을 오픈소스로 공개 → 지자체별 브랜드 커스터마이징
  용역 수요 창출.
- **복지관 B2G**: 경로당·복지관 관리자용 대시보드(프로그램 예약·참여자 관리) 유료
  모듈로 확장.
- **시니어 디지털 튜터**: 앱 사용 안내를 위한 60대 이상 '디지털 봉사단' 양성 —
  과기정통부 디지털배움터 사업과 연계 [2].

### 6.3 정량 목표 (론칭 후 6~12개월)

| 지표 | 6개월 목표 | 12개월 목표 |
|---|---|---|
| 월 활성 이용자 (MAU) | 3만 | 15만 |
| 누적 외출 체크인 | 15만 건 | 120만 건 |
| 보호자 공유 링크 활성 수 | 1만 | 8만 |
| 추천 카드 클릭 전환율 | 25% | 40% |
| 연계 경로당·쉼터 수 | 수도권 1,000개소 | 전국 10,000개소 |

### 6.4 공공가치 선순환

> **더 많은 외출 체크인 → 더 정확한 "오늘 갈 곳" 추천 → 더 높은 만족도
> → 더 많은 이용자 → 더 풍부한 데이터**

이 데이터 선순환은 초고령사회 공공자원 연결의 모범 사례로 자리잡는다.
