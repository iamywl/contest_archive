## 1. 개발 결과물 개요

**책크인**은 행정안전부·한국지역정보개발원이 2025년부터 전국 단위로 통합 개방한
**공공도서관 열람실 현황**, **전국 공영자전거**, **공영 물품보관함**의 실시간 API를
하나의 모바일 웹 지도에서 교차 연결하고, AI가 좌석 혼잡도 예측과 개인 맞춤
이동·학습 동선을 제시하는 **MZ세대 학습 루틴 앱**이다 [1][2].

이용자는 "지금 자리가 있는 가장 가까운 도서관"을 실시간으로 탐색하고, 공영자전거로
이동하여 주변 보관함에 짐을 맡긴 뒤 도착과 동시에 체크인·타이머로 학습 시간을
자동 기록한다. 이 모든 과정이 **단일 앱의 한 플로우**로 묶인다는 점이 각 기관별
홈페이지·개별 앱에 흩어진 기존 서비스와의 결정적 차별점이다.

**한 줄 요약**: "스터디카페를 대체하는 무료 공공자원 기반 학습 루틴 앱."

## 2. 개발 결과물의 기능 및 특징

### 2.1 서비스의 목적 및 배경

#### 문제 정의

- **스터디카페 시장은 폭발적으로 성장, 학습비는 고정비화됐다.**
  2015년 112개에 불과했던 스터디카페 가맹점 수는 2024년 10월 기준 6,944개로 10년간
  약 62배 증가했고, 전국 추정 점포 수는 12,000개 이상이다 [3]. 결제 고객 중 20대
  비중 약 30%, 중·고등학생 자녀의 이용권을 부모가 결제한 매출도 전체의 32%에
  달해 학생·청년·학부모 모두에게 지속적 비용 부담이 되고 있다 [3].

- **반면, 공공도서관의 유휴 자원은 막대하다.**
  2024년 말 기준 전국 공공도서관은 1,296개관, 연간 방문자 1관당 17만 3천 명에
  달하며 접근성·인프라 측면에서 스터디카페의 공공 대안이 충분하다 [4][5]. 그러나
  **좌석 정보·주변 이동수단·짐 보관 정보가 기관별·서비스별로 분산**되어 실질적인
  대체 활용이 어렵다.

- **따릉이 등 공영자전거 생태계는 이미 일상이 됐다.**
  서울 공공자전거 '따릉이' 는 2024년 한 해에만 4,385만여 건이 이용됐고, 지난 10년
  누적 2.5억 건을 기록했다 [6][7]. 단거리 생활형 이동수단으로 자리잡은 만큼, 학습
  동선과 자연스럽게 결합할 수 있다.

- **MZ세대 학습자는 "한 화면 원스톱" UX를 원한다.**
  어디로 갈지, 어떻게 갈지, 짐은 어떻게 할지, 얼마나 공부할지를 **한 번의 탐색**으로
  결정하고자 한다. 현재 공공서비스는 이 단일 플로우에 대응하지 못한다.

#### 해결 방향

1. **전국 통합개방 실시간 API 3종을 단일 플로우로 통합**한다 [1][2].
2. **AI 예측**으로 "1시간 뒤 자리 있을까?"라는 미래 시점의 불확실성을 제거한다.
3. **LLM 기반 자연어 코치**가 "오늘 비 오는데 어디 가야 해?" 같은 복합 조건을
   공공데이터 Tool-use로 답한다 [8][9].
4. **체크인·타이머·주간 리포트**로 공공자원 기반 학습 습관을 정량화한다.

### 2.2 서비스의 핵심 기술·기능·구성·특징

#### (1) 5대 핵심 화면

| # | 화면 | 핵심 기능 |
|:--:|---|---|
| ① | **홈 (지도 + 리스트)** | 내 위치 기반 공공도서관 마커를 좌석 여유도 3단계(🟢 여유 / 🟡 보통 / 🔴 혼잡) 색상 코딩. 거리순·여유순 정렬 리스트 동시 제공. 5분 자동 갱신. |
| ② | **도서관 상세** | 실시간 총·점유·잔여 좌석, 열람실별 세부, **AI 혼잡도 예측 그래프(향후 3시간)**, 가까운 공영자전거 3곳, 주변 보관함 3곳을 한 화면 스크롤로 제공. 하단 고정 "체크인" CTA. |
| ③ | **AI 학습 코치** | 자연어 질의응답. 사용자 위치·시각·날씨 맥락을 파악해 실시간 공공데이터를 **Tool-use**로 조회하고 도서관·자전거·보관함을 묶어 문장형 추천 [8]. |
| ④ | **체크인** | GPS/QR 체크인, 실시간 공부 타이머, 체크아웃 시 기록 자동 저장. |
| ⑤ | **내 루틴** | 최근 7일 학습시간 막대그래프, 연속일(🔥 스트릭), 체크인 이력, 누적 통계. |

#### (2) AI 기능 (공모전 가점 10점 대상) [8][9]

- **좌석 혼잡도 단기 예측 모듈 `/api/ai/forecast`**
  시간대별 점유 패턴과 현재 잔여좌석을 입력으로 향후 3시간의 잔여좌석 곡선을
  반환한다. 초기 서비스에는 휴리스틱 + 이동평균 모델로 출시하고, 체크인 로그가
  축적되면 Prophet·LightGBM·LSTM 회귀 등으로 고도화한다 [10][11].

- **자연어 학습 코치 `/api/ai/chat`**
  LLM Tool-use 패턴을 활용한다. LLM은 다음 도구를 필요 시 호출한다:
  `search_libraries(near, minSeats)` → `nearby_bikes(lat, lng)` →
  `nearby_lockers(lat, lng)` → `get_forecast(lib_id)` → `weather_now()`. "비 오는데
  조용하고 자리 많은 곳" 같은 복합 조건을 파싱해 **실제 공공데이터 결과를 근거로**
  답한다 [8].

- **주간 학습 리포트 자동 생성** *(Phase 2)*
  LLM이 사용자 체크인 로그를 요약해 "이번 주 집중시간은 화요일 오전이 가장
  길었어요", "다음 주 목요일엔 강남도서관이 비교적 한산할 전망" 같은 개인화
  인사이트를 생성한다.

#### (3) 시스템 구성

```
[Frontend: Next.js 16 App Router · TypeScript · Tailwind CSS v4]
          ↓
[Backend API Routes — 공공데이터 프록시 + 5분 TTL 캐시]
  ├─ /api/libraries                         (③ 공공도서관 열람실)
  ├─ /api/bikes?near=lat,lng                (① 전국 공영자전거)
  ├─ /api/lockers?near=lat,lng              (④ 공영 물품보관함)
  ├─ /api/ai/forecast?lib=id                (AI 예측)
  └─ /api/ai/chat                           (LLM 에이전트)
          ↓
[공공데이터포털 API] [LLM API] [Supabase DB (Phase 2)]
```

- **공공 API 프록시 레이어**: 인증키를 서버 측에 보관하여 유출을 방지하고, 5분 TTL
  인메모리 캐시 + **stale-while-revalidate** 로 상류 API 장애 시에도 서비스
  연속성을 유지한다.
- **지도**: Leaflet + OpenStreetMap(무료·API 키 불필요)으로 심사·시연 환경 안정성
  확보.
- **로컬 저장**: 비회원 체크인도 즉시 체험 가능하도록 Zustand + localStorage persist
  사용.

## 3. 기존 서비스와 차별성 및 독창성

### 3.1 기존 유사 서비스 비교

| 항목 | 각 지자체 도서관 홈페이지 | 따릉이 앱 | 민간 스터디카페 앱 | **책크인** |
|---|:---:|:---:|:---:|:---:|
| 실시간 좌석 조회 | ⚠️ (기관별 개별) | ❌ | ❌ | **✅ (전국 통합)** |
| 공영자전거 경로 연동 | ❌ | ✅ | ❌ | **✅** |
| 공영 물품보관함 현황 | ❌ | ❌ | ❌ | **✅** |
| 좌석 혼잡도 예측 | ❌ | ❌ | ❌ | **✅ (AI)** |
| 자연어 학습 코치 | ❌ | ❌ | ❌ | **✅ (LLM)** |
| 학습 루틴 기록 | ❌ | ❌ | 유료 부가서비스 | **✅ (무료)** |
| 월 평균 이용료 | 무료 | 저가 (정기권) | 약 15만원 [3] | **무료** |

### 3.2 독창적·창의적 요소

1. **공공자원 체인 라우팅 (Chained Public-Resource Routing)**
   단일 목적 정보 제공을 넘어 **"도서관 좌석 + 자전거 + 보관함"을 시공간 체인으로
   묶는 동선 설계**. 국가 MaaS(K-MaaS) 전략이 교통수단 간 연계에 주력하는 것과
   달리, 본 서비스는 **학습 자원까지 MaaS의 대상으로 확장**한다는 점에서 독창적이다
   [12][13].

2. **실시간 공공데이터 × 시계열 예측의 결합**
   실시간 API만으로는 "지금 여기 자리가 있는가"만 파악할 수 있다. 책크인은
   **30분·1시간·3시간 뒤의 전망**을 얹어 이용자의 이동 결정을 돕는다. 의료(응급실
   혼잡도 예측), 교통(도로 속도 예측) 분야에서 검증된 시계열 딥러닝 패러다임을
   **공공 학습 인프라에 최초로 적용**한다 [10][11].

3. **LLM Tool-use 기반 공공데이터 에이전트**
   단순 챗봇이 아닌, LLM이 공공 API를 도구로 호출하여 **실사실(Ground-Truth) 기반
   응답**을 생성하는 에이전트 아키텍처. 법제처 국가법령정보 MCP 등과 같이 공공
   데이터 서비스의 AI 에이전트화가 가속되는 흐름과 맞닿아 있다 [9].

4. **체크인–자원 선순환 피드백 루프**
   이용자의 체크인 로그가 혼잡도 예측 모델의 학습셋이 되어, 서비스 성장이
   AI 품질 향상으로 직결되는 **폐쇄형 개선 구조**. 시간이 흐를수록 정확도가 자동
   상승한다.

### 3.3 지식재산권 전략

- "실시간 공공자원 통합 좌석 예측 시스템 및 방법" 비즈니스모델 특허 출원 검토.
- 핵심 프록시·캐시 레이어는 MIT 라이선스 오픈소스로 공개하여 지자체 확산을 유도한다.

## 4. 공공데이터 등 데이터 활용성

### 4.1 필수 활용 공공데이터 (전국 통합개방데이터 7종 중 3종)

본 서비스가 활용하는 3종은 모두 **2025년 5월 행정안전부·한국지역정보개발원이
통합 개방한 데이터**로, 공공데이터포털에서 오픈 API 형태로 제공된다 [1][2].

| 데이터명 | API 종수 | 원천 기관 | 획득 방법 | 서비스 내 활용 |
|---|:---:|---|---|---|
| **③ 공공도서관 열람실 현황 실시간 정보** | 3종 | 한국지역정보개발원 (전국 통합) [14] | 공공데이터포털 인증키 발급 후 REST API 폴링 (5분 주기) | 홈 지도 마커, 좌석 수치, 열람실별 세부 현황, AI 예측 모델 입력 |
| **① 전국 공영자전거 실시간 정보** | 3종 | 서울·대전·세종·광주·창원·여수 등 11개 지자체 통합 [2] | 동일 (2분 주기) | 도서관 주변 대여소·이용가능대수, 이동 경로 제안 |
| **④ 공영 물품보관함 현황 실시간 정보** | 3종 | 지자체 통합 | 동일 (5분 주기) | 도서관 도보 반경 내 빈 보관함 수, 추천 |

### 4.2 타 기관 공공·민간 데이터 보조 활용

| 데이터 | 출처 | 획득 방법 | 활용 목적 |
|---|---|---|---|
| 단기 기상 예보 | 기상청 API허브 | REST | 비·폭염·한파 시 실내 수요 폭증 보정 (예측 정확도 향상) |
| 도서관 장서 검색 | 문화체육관광부 도서관 정보나루 [15] | Open API | Phase 2: 특정 도서 소장·대출가능 여부 연계 |
| 지도 타일 | OpenStreetMap | 공개 타일 서버 | 지도 시각화 (API 키 불필요) |

### 4.3 공공데이터 획득의 지속성

- 활용 API는 모두 공공데이터포털 **상시 개방 국가중점데이터**로 지속성이 제도적으로
  보장된다. 2024년 말 기준 공공데이터 개방 건수는 10만 건, 국가중점데이터 217종에
  달한다 [16].
- 인증키 만료·정기 순단에 대비해 **5분 TTL 인메모리 캐시 + stale-while-revalidate**
  전략으로 상류 API 단기 장애에 강건하게 설계했다.
- 데이터 갱신 주기(좌석 5분, 자전거 2분, 보관함 5분)를 존중해 과도한 호출로 인한
  Rate Limit 초과를 방지한다.

### 4.4 데이터 가공 및 결합

- 세 데이터는 **위경도(Geo) 속성을 공통 키**로 결합한다. Haversine 거리 기반
  매칭, 5분 단위 시공간 조인으로 "한 화면 동선"을 구성한다.
- 좌석 데이터는 시간 축을 따라 누적 저장하여 **시간대별 점유 패턴 학습셋**을
  구축한다 (Phase 2 예측 고도화 입력).
- 모든 화면과 API 응답에 **공공데이터 출처를 명시**한다 (공고 요건 준수).

## 5. 개발 결과물의 기술성과 구현 및 실현가능성

### 5.1 활용한 주요 기술

| 레이어 | 기술 | 선정 사유 |
|---|---|---|
| 프레임워크 | Next.js 16 App Router + TypeScript | SSR 초기 로딩, Vercel 즉시 배포, 서버/클라이언트 컴포넌트 분리 |
| 스타일 | Tailwind CSS v4, Lucide Icons | 빠른 프로토타이핑, 일관된 디자인 토큰 |
| 지도 | Leaflet + OpenStreetMap | 무료·API 키 불필요 → 시연 안정성 |
| 상태 관리 | TanStack Query (서버 캐시) / Zustand + persist | 자동 리페치, 비회원 체크인 영속화 |
| 차트 | Recharts | 반응형, 모바일 터치 지원 |
| AI | OpenAI GPT-4o-mini / Anthropic Claude Haiku 선택형 (Tool-use) [8] | 저비용, 구조화된 함수 호출 지원 |
| DB (Phase 2) | Supabase (PostgreSQL) | 인증·RLS·실시간 내장 |
| 배포 | Vercel | 무료 티어, GitHub 연동 자동배포 |

### 5.2 구현 상태 (본 기획서 제출 시점)

**시연 가능한 데모 버전이 이미 완성되어 있다.**

- 5개 MVP 화면 동작 (홈 · 도서관상세 · AI코치 · 체크인 · 내루틴)
- 공공데이터 3종 프록시 API + AI 2종(예측·챗봇) **총 7개 엔드포인트** 구동
- TypeScript 타입 검증 및 **프로덕션 빌드 통과**
- 실제 서울 도서관·따릉이·보관함 좌표 기반 12개 도서관 시연 셋 탑재
- 입력(자연어 질의) → 처리(공공API 조회·거리 계산·LLM 추론) → 출력(통합 추천)의
  **완전한 엔드투엔드 플로우** 시연 가능

**제출물**: Vercel 배포 URL + 접속 QR 코드 + 화면 녹화 영상.

### 5.3 실현 가능성 평가

| 리스크 | 영향도 | 대응 |
|---|:---:|---|
| 공공 API 일시 장애 | 높음 | 5분 TTL 캐시 + stale-while-revalidate + mock 폴백 |

| 이용자 유입 저조 | 중간 | QR 단순화, 친구 초대·랭킹, 지자체 협력 |

## 6. 기대 효과

### 6.1 사회적 가치 창출

- **교육비 부담 경감 (직접 효과)**
  스터디카페 월 평균 이용료 약 15만원 [3] → 공공도서관 무료 대체.
  가구당 **연 최대 180만원** 절감 가능. 청년·저소득·수험생 계층의 학습 고정비 감소.

- **공공자원 활용률 제고 (공공 가치)**
  공공도서관은 2024년 1관당 연간 방문자 17만 3천명 수준이지만 [4], **유휴 좌석에
  대한 검색·매칭 경로**는 부재하다. 본 서비스는 유휴 좌석과 학습자를 실시간으로
  매칭해 **이용률 15% 이상 상승**을 목표로 한다 (서비스 론칭 1년 내).

- **탄소 저감 (환경 가치)**
  따릉이는 2024년 4,385만 건 이용으로 이미 도시 단거리 이동의 일상 수단이다 [6].
  책크인은 **학습 동선에 공영자전거를 자연스럽게 결합**하여 연 50만 건 이상의
  친환경 단거리 이동을 유도한다.

- **지역 격차 완화**
  본 서비스는 수도권·대도시 중심의 민간 스터디카페 대신 **전국 공공도서관의
  동등한 노출**을 제공한다. 현재 1,296개관 중 다수가 지방에 있으나 [4] 접근 정보의
  디지털 격차 때문에 활용이 저조하다. 통합 지도 UX가 이 격차를 해소한다.

### 6.2 상생협력·일자리 창출 효과

- **지자체 확산 모델**: 핵심 로직을 오픈소스로 공개 → 지자체별 브랜드 커스터마이징
  용역 수요 창출. 중복 개발 예산의 효율적 재배분.
- **대학·도서관 제휴 B2G 모델**: 학생증 연동·장서 검색·연속출석 리워드 등 제휴형
  확장 서비스 개발 → 에듀테크 스타트업의 신규 수익 모델.
- **청년 개발자 참여**: 공공데이터 기반 오픈소스 프로젝트로 청년 개발자 커뮤니티의
  포트폴리오·레퍼런스 자산.

### 6.3 정량 목표 (론칭 후 6~12개월)

| 지표 | 6개월 목표 | 12개월 목표 |
|---|---|---|
| 월 활성 이용자 (MAU) | 3만 | 10만 |
| 누적 체크인 수 | 20만 건 | 80만 건 |
| 공영자전거 유도 이용 | 5만 건 | 50만 건 |
| AI 추천 적중률 (체크인 전환) | 25% | 40% |
| 연계 도서관 수 | 서울 100관 | 전국 500관 |
| 사용자 평균 주간 학습시간 | 주 5시간 | 주 8시간 |

### 6.4 공공가치 선순환

> **더 많은 이용자 → 더 많은 체크인 데이터 → 더 정확한 AI 예측
> → 더 높은 만족도 → 더 많은 이용자**

이 데이터 선순환은 공공 인프라의 효용을 AI 기술이 증폭시키는 모범 사례로 자리잡아,
**공공데이터 개방 정책의 성공적 활용 레퍼런스**가 된다.
