## 1. 기술 스택

### 1.1 프론트엔드

| 레이어 | 기술 | 버전 | 선정 사유 |
|---|---|---|---|
| 프레임워크 | **Next.js** (App Router) | 16.2.4 | SSR 초기 로딩, 서버 컴포넌트, Vercel 즉시 배포 |
| 언어 | **TypeScript** | 5.9 | 타입 안정성, 공공데이터 스키마 검증 |
| 번들러 | **Turbopack** | Next 내장 | 빠른 HMR |
| 스타일 | **Tailwind CSS** | 4.2 | 디자인 토큰 일관성 |
| 아이콘 | **Lucide React** | 1.x | 경량, 모바일 최적 |
| 지도 | **Leaflet + react-leaflet** | 1.9 / 5.0 | 무료·API 키 불필요 |
| 타일 | **OpenStreetMap** | — | 공개 타일 서버 |
| 차트 | **Recharts** | 3.x | 반응형 SVG 차트 |

### 1.2 상태 관리

| 영역 | 기술 | 역할 |
|---|---|---|
| 서버 상태 | **TanStack Query** 5.x | 공공데이터 fetch · 캐시 · 리페치 |
| 클라이언트 상태 | **Zustand** 5.x + `persist` | 체크인 세션 · 루틴 이력(localStorage) |

### 1.3 백엔드 (Next.js Route Handlers)

| 엔드포인트 | 역할 |
|---|---|
| `GET /api/libraries` | 공공도서관 열람실 실시간 정보 프록시 |
| `GET /api/libraries/[id]` | 단일 도서관 조회 |
| `GET /api/bikes?near=lat,lng` | 전국 공영자전거 주변 탐색 |
| `GET /api/lockers?near=lat,lng` | 공영 물품보관함 주변 탐색 |
| `GET /api/ai/forecast?lib=id` | AI 좌석 혼잡도 단기 예측 |
| `POST /api/ai/chat` | LLM Tool-use 기반 통합 추천 |

### 1.4 AI·데이터

| 영역 | 기술 | 비고 |
|---|---|---|
| LLM | OpenAI GPT-4o-mini / Anthropic Claude Haiku | Tool-use 지원, 데모는 규칙 기반 |
| 예측 모델 | 휴리스틱 + 이동평균 → (Phase 1) Prophet / LightGBM | 체크인 로그 축적 후 교체 |
| DB | Supabase PostgreSQL (Phase 2) | 인증·RLS·실시간 |
| 캐시 | 인메모리 5분 TTL + stale-while-revalidate | `src/lib/data/cache.ts` |
| 공공API | 공공데이터포털 | `DATA_GO_KR_KEY` 미설정 시 mock fallback |

### 1.5 배포·운영

| 영역 | 기술 | 비고 |
|---|---|---|
| 호스팅 | Vercel | Next.js 공식, 무료 티어 |
| 도메인 | Vercel 서브도메인 (예정) | 커스텀 도메인 Phase 2 |
