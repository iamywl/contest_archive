# 03. 시연 영상 녹화

**목표**: 책크인의 핵심 기능을 **1~2분 내**로 보여주는 시연 영상을 만든다.
**예상 소요**: 30분~1시간 (리테이크 2~3회 포함)

---

## 0. 왜 짧게 찍는가

심사위원은 10개 팀 이상의 영상을 연달아 봅니다.
첫 **15초** 안에 "무엇을 해결하는가"가 드러나지 않으면 다음 페이지로 넘어갑니다.

**이상적 길이**: 1분 30초 (최대 2분).

---

## 1. 장비·도구

### 1.1 최소 세트 (macOS)

- **녹화**: macOS 기본 `Cmd + Shift + 5` → 화면 녹화
- **편집**: iMovie (기본 설치) — 자막·잘라내기·음성 overlay
- **나레이션 마이크**: 맥북 내장 마이크로 충분. 정숙한 방에서 녹음.

### 1.2 더 나은 품질이 필요하면 (선택)

- [OBS Studio](https://obsproject.com/) (무료) — 화면 + 웹캠 동시 녹화
- [Loom](https://www.loom.com/) — 링크로 바로 공유, 자동 캡션
- [Screen Studio](https://screen.studio/) — macOS 전용, 커서 강조·줌 자동

---

## 2. 시연 시나리오 (대본)

전체 1분 30초 기준. 각 장면의 **나레이션 스크립트**는 영상 자막으로도 사용.

### 장면 1 — 문제 제기 (0:00 ~ 0:15)

**화면**: 검은 배경에 텍스트 인트로 또는 스터디카페 요금표 이미지.
**나레이션**:
> "스터디카페 월 15만원. 연 180만원.
> 전국 1,296개 공공도서관의 유휴 좌석을 두고도
> 정보가 흩어져 있어 이용하지 못합니다."

**자막 포인트**: `"월 15만원" "공공도서관 1,296개" "정보가 흩어져 있다"`

### 장면 2 — 홈 지도 (0:15 ~ 0:30)

**화면**: `https://chekcin.vercel.app/` 접속 → 위치 권한 허용 → 지도에 도서관 마커.
**나레이션**:
> "책크인. 지금 내 주변 공공도서관의 실시간 좌석을
> 지도에서 한눈에 봅니다.
> 초록은 여유, 빨강은 만석."

**조작**: 지도 드래그·줌 한 번, 리스트에서 도서관 하나 클릭.

### 장면 3 — 도서관 상세 + AI 예측 (0:30 ~ 0:55)

**화면**: 도서관 상세에서 실시간 좌석 → **AI 혼잡도 예측 그래프** → 가까운 따릉이 → 보관함.
**나레이션**:
> "상세 화면에서는 AI가 향후 3시간 좌석 전망을 그려주고,
> 가장 가까운 따릉이 대여소와 도서관 앞 보관함까지
> 한 번에 확인할 수 있습니다."

**강조 포인트**: AI 예측 곡선 위에 마우스로 트래킹 (커서 강조 효과).

### 장면 4 — AI 학습 코치 (0:55 ~ 1:15)

**화면**: AI 코치 탭 → 추천 질문 "비 오는데 가깝고 자리 많은 데 어디야?" 클릭.
**나레이션**:
> "자연어로 물어보면 AI가
> 공공 API 세 가지를 직접 조회해
> 도서관, 따릉이, 보관함을 묶어 추천합니다."

**포인트**: 응답이 나오는 3초 정도 기다려서, **실데이터 기반** 이라는 인상을 강조.

### 장면 5 — 체크인 → 루틴 (1:15 ~ 1:25)

**화면**: 도서관 상세에서 체크인 → 타이머 → 체크아웃 → 루틴 대시보드 에 기록.
**나레이션**:
> "체크인하면 공부 시간이 자동 기록되고
> 주간 학습 루틴으로 쌓입니다."

> 💡 실제 녹화 시 타이머를 기다리지 말고 **컷 편집**으로 체크인 → 10초 건너뛰기 → 체크아웃.

### 장면 6 — 마무리 (1:25 ~ 1:30)

**화면**: 로고 + URL + QR 코드.
**나레이션**:
> "공공데이터로 만드는 무료 학습 루틴,
> 책크인."

## 3. 녹화 전 환경 세팅

### 3.1 브라우저 준비 (크롬/사파리)

- 모든 개인 탭 닫기 · 확장 프로그램 아이콘 가리기
- 전체화면 모드 또는 **시크릿 창** 사용
- 북마크바 숨기기: `Cmd + Shift + B`
- 주소창은 클린하게: 접속할 URL만 미리 열어둠
- 알림 차단: 슬랙·디스코드·카톡 전부 DND (방해금지) 모드

### 3.2 iPhone 미러링 (더 임팩트 있음)

모바일 앱 느낌을 주려면 실제 iPhone 을 녹화:

```bash
# iPhone 을 Lightning/USB-C 로 Mac 연결
# QuickTime → File → New Movie Recording → 녹화 버튼 옆 ▼ → 카메라·마이크: iPhone 선택
```

또는 시뮬레이터:
```bash
# Xcode 설치 상태에서
open -a Simulator
# 시뮬레이터에서 Safari → URL 입력
```

### 3.3 책크인 앱 사전 상태

- localStorage 초기화 (이전 체크인 기록 없는 상태에서 시작)
- 위치 권한은 **미리 허용**해둠 (녹화 중 팝업 피하기)

초기화 방법:
```js
// 브라우저 콘솔에서
localStorage.clear(); location.reload();
```

## 4. 녹화 실행 (macOS)

### 4.1 Cmd+Shift+5 로 간단히

1. `Cmd + Shift + 5`
2. 툴바에서 **[Record Selected Portion]** 선택
3. 녹화 영역 지정 (브라우저 창 전체 or 필요 부분)
4. **[Record]** 클릭 → 녹화 시작
5. 중지: 메뉴바 우측 상단의 녹화 중지 아이콘 or `Cmd + Ctrl + Esc`

### 4.2 OBS 로 고급 녹화

1. Scenes → `+` → "책크인 시연"
2. Sources → `+` → `Window Capture` → 브라우저 창 선택
3. Sources → `+` → `Audio Input Capture` → 마이크
4. Settings → Output → Recording Format: `mp4`, Encoder: `Apple VideoToolbox H.264`
5. **Start Recording**

## 5. 편집 (iMovie 기준)

### 5.1 프로젝트 만들기

1. iMovie → **Create New → Movie**
2. 녹화한 mp4 파일 import (Drag & Drop)
3. 타임라인에 올리기

### 5.2 필수 편집 3가지

1. **잘라내기**: 불필요한 대기시간 제거 (타이머 기다리기 등). 클립 선택 → `Cmd + B` 로 분할 → 삭제.
2. **자막**: Titles 탭 → **Basic Lower Third** → 텍스트 입력. 장면별 핵심 키워드만.
3. **속도 조절**: 조작이 느린 부분 1.5~2배속. 클립 선택 → `Speed` 슬라이더.

### 5.3 인트로·아웃트로 (선택)

- Titles 탭의 **Centered** 템플릿으로 "책크인 — Check-In to Read" 0.5초 인트로.
- 마지막에 URL + QR 이미지 3초.

### 5.4 BGM (선택)

저작권 free 한 배경음:
- YouTube Audio Library: https://www.youtube.com/audiolibrary
- 나레이션 있으면 볼륨 20~30% 로 낮춰서.

## 6. 내보내기

### 6.1 iMovie

1. File → Share → File
2. Resolution: **1080p** (4K는 파일 너무 큼)
3. Quality: High
4. Compress: Faster
5. mp4 파일 저장 → `docs/screenshots/demo.mp4`

### 6.2 파일 크기 확인

50~150MB 정도가 적당. 200MB 초과 시 재인코딩:

```bash
# ffmpeg 설치돼 있으면
ffmpeg -i demo.mp4 -vcodec libx264 -crf 28 -preset medium demo-compressed.mp4
```

## 7. 업로드·공유

### 7.1 YouTube (Unlisted 추천)

1. https://studio.youtube.com → 업로드
2. **Visibility: Unlisted** (링크 있는 사람만, 검색 안 됨)
3. 업로드 후 링크 복사
4. 기획서 부록에 기재

### 7.2 Google Drive / Dropbox

1. 드라이브 업로드 → 공유 링크 (링크 가진 모두 볼 수 있음)
2. 권한: **Viewer**

### 7.3 기획서 부록에 삽입

```markdown
### 부록. 시연 영상
- URL: https://youtu.be/XXXXXXX
- 길이: 1분 30초
- QR 코드: ![영상 QR](demo-video-qr.png)
```

## 8. 품질 체크리스트

녹화 완료 후 본인이 먼저 리뷰:

- [ ] **길이**: 1~2분 이내?
- [ ] **첫 15초**: 문제·해결이 명확?
- [ ] **AI 기능** 2가지(예측·챗봇) 모두 노출?
- [ ] **공공데이터 3종**(도서관·자전거·보관함) 모두 화면에 나옴?
- [ ] **소리**: 나레이션 명확? BGM 너무 크지 않음?
- [ ] **커서**: 화면 어디를 보는지 명확?
- [ ] **끊김**: 로딩 지연·에러 화면 안 보임?
- [ ] **결제 창·알림·개인정보** 가 화면에 없음?

## 9. 자주 발생하는 문제

| 증상 | 해결 |
|---|---|
| 녹화 파일이 `.mov` 로 나와 용량 큼 | `ffmpeg -i input.mov -c:v libx264 output.mp4` |
| 소리가 안 녹음됨 | 시스템 환경설정 → 보안 → 마이크 → 녹화 앱 허용 |
| iPhone 화면 선명도 낮음 | QuickTime → Quality: High, 또는 iOS 네이티브 녹화 후 AirDrop |
| 지도 타일이 안 뜸 | Leaflet OSM 타일 로딩 지연. 녹화 전 2~3초 대기 |
| AI 응답 너무 느림 | 실 LLM 쓸 때 느릴 수 있음. 녹화 시 mock 모드로 촬영하거나 편집에서 잘라냄 |
