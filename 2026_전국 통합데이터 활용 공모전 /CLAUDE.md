# 2026 전국 통합데이터 활용 공모전 — 작업 지침

이 저장소는 **행정안전부·한국지역정보개발원 주관 2026 전국 통합데이터 활용 공모전** 출품을 위한 작업 공간입니다. AI 어시스턴트는 아래 원칙과 산출물 규격을 엄격히 따릅니다.

---

## 디렉터리 규칙

```
/
├── CLAUDE.md          # (이 문서) 작업 지침
├── docs/              # 문서용 산출물 — 모두 Markdown
│   ├── 제안서.md
│   ├── 개발계획서.md
│   ├── 개발보고서.md
│   ├── 검증보고서.md
│   ├── 설문조사.md
│   └── screenshots/
├── dev/               # 개발용 산출물
│   └── chekcin/       # Next.js 데모 앱
├── todo/              # 사용자 수행 체크리스트 (API키·배포·녹화 등)
└── (공고문 원본 hwpx)
```

- **문서용(`docs/`)과 개발용(`dev/`)을 섞지 않는다.**
- 새 문서는 반드시 Markdown(`.md`)으로 작성한다. 필요 시 외부에서 hwpx/PDF 변환.

## 문서별 규격

### 1) `docs/제안서.md`

- **목적**: 공모전 심사용 [붙임4] 서비스 개발 기획서의 본문. 공고 지정 6개 섹션 순서 준수.
- **핵심 요구사항**
  - **참고문헌(References)을 반드시 포함**한다.
  - 참고문헌은 **철저히 조사**하여 신뢰할 수 있는 정부·공공·학술·언론 출처를 인용한다. 출처별 URL/발행기관/발행연도 명시.
  - 본문의 수치·주장은 가능한 한 각주 또는 괄호 인용으로 출처 번호와 연결한다 (예: `(KOSIS, 2024)` 또는 `[1]`).
  - 10페이지 내외 분량, 팀원 개인정보는 포함하지 않는다.
- **섹션 순서(공고 고정)**
  1. 개발 결과물 개요
  2. 개발 결과물의 기능 및 특징
  3. 기존 서비스와 차별성 및 독창성
  4. 공공데이터 등 데이터 활용성
  5. 개발 결과물의 기술성과 구현 및 실현가능성
  6. 기대 효과
- **마지막**: `## 참고문헌` 섹션 필수.

### 2) `docs/개발계획서.md`

- **목적**: 개발 일정·상태를 팀 내부와 심사위원에게 한눈에 보여주는 문서.
- **핵심 요구사항**
  - **표 위주**로 기술한다. 장문 서술 최소화.
  - **기술 스택 표**를 포함한다 (레이어·기술·선정사유).
  - **개발 일정 표**: 언제부터(Start) ~ 언제까지(End) + **현재 상태(Status)** 열 포함. 상태값은 `✅ 완료 / 🟡 진행중 / ⬜ 예정 / ⚠️ 지연`.
  - **현재 진척도 요약**(% 또는 완료/전체 작업 수)을 문서 상단에 배치.
  - 마일스톤·리스크·대응도 표로 정리.

### 3) `docs/개발보고서.md`

- **목적**: 실제 구현 결과물의 동작을 증거와 함께 보고하는 문서.
- **핵심 요구사항**
  - **실제로 구현된 소프트웨어의 화면을 캡처**하여 첨부한다. 캡처는 `docs/screenshots/` 디렉터리에 저장하고 Markdown 이미지 링크로 삽입한다.
  - **소프트웨어를 실제로 구동**한 뒤 캡처한다 (모의 목업 금지). 구동 환경(포트·브라우저·해상도)을 명시한다.
  - **캡처된 사진을 보고서 작성 과정에서 직접 검토**한다. 보이는 결함·어색한 UI·의도와 다른 부분을 식별한다.
  - **미흡하거나 의도와 다르면 즉시 수정**한 뒤 **재캡처**하여 반영한다. 수정 이력은 보고서에 기록한다.
  - 각 캡처 아래에는 "무엇을 보여주는가 / 의도 / 검토 결과 / 조치" 형식의 설명을 붙인다.

## 코드 및 커밋 원칙

- `dev/chekcin/` 내 Next.js 앱은 프로덕션 빌드(`pnpm build`)가 항상 통과해야 한다.
- 데모 안정성을 위해 공공 API 키가 없을 때도 mock fallback으로 동작해야 한다.
- UI 변경 시 반드시 `pnpm dev` 실행 → 스크린샷 캡처 → 개발보고서에 반영.
- 스크린샷 캡처 자동화 스크립트는 `dev/chekcin/scripts/` 하위에 둔다.

## 글쓰기 원칙

- 한국어로 작성한다. 용어는 공모문(공고문)의 표기를 따른다.
- 배점이 높은 항목(참신성 30, 공공데이터 활용성 25, 국민편익 20)에 살을 붙인다.
- AI 가점(+10) 요건은 2군데 이상 명시한다.
- 근거 없는 수치·단정은 쓰지 않는다. 수치를 쓰면 반드시 출처를 명기한다.
