## 공통 작업 원칙

각 공모전 디렉터리 하위에 지침 파일이 있다. 문서 규격, 캡처 의무, 코드 표준 등이 정리되어 있으며, 새 작업을 시작하기 전에 반드시 먼저 읽는다.

- 모든 문서는 Markdown(`.md`)로 작성한다. hwpx/PDF 변환은 최종 제출 단계에서만 한다.
- 제안서류 수치·주장은 1차 출처(정부 통계·공공기관 발표·학술 논문) 각주로 뒷받침한다.
- 보고서는 **실제 구동된 소프트웨어 캡처**로 증거를 남긴다 (목업 금지).
- API 키·시크릿은 `.env` 파일로 분리하며 저장소에 커밋하지 않는다.
