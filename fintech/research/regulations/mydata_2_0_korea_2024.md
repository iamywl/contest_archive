# 마이데이터 2.0 추진방안 (2024-04)

## 배경: 기존 마이데이터의 한계
- **서비스 차별성 부족**: 표준 API 동일 데이터 → "종합자산관리·상품추천" 대동소이. ([율촌](https://yulchonllc.com/fix/2024/fix-DF-2407/PDF/2407_YC_DF_NL_10_Sub03.pdf))
- **정보 제공 불완전**: 장기 미사용 계좌 등 조회 곤란. ([ifs.or.kr](https://ifs.or.kr/bbs/board.php?bo_table=News&wr_id=53393))
- **정보보안 사고**: 초기 일부 사업자에서 타인 정보 조회·제3자 판매 정황 발생.
- **사업자 수익성 악화**: 2024년 과금 본격 시행에 따른 중소 사업자 위협.
- **겸영·확장 제약**: 오프라인·비금융 겸영 제한으로 신규 서비스 출시 곤란.

## 2024 개선 방향
- 정보보안 강화.
- 겸영업무 유연화.
- 오프라인 가입 허용.
- 정보 결합 기준 명확화.
- 중복 API 호출 과금 합리화 등.
- 추가로 **전금법 등 연내 개정** 추진 보도. ([디지털데일리 2024-04](https://m.ddaily.co.kr/page/view/2024040416465090630))

## 본 프로젝트 관련 시사점
- 표준 데이터로는 서비스 차별성을 내기 어렵다는 구조적 한계 → **추천·시뮬레이션의 품질·설명성에서 차별화**가 핵심 포지션.
- 2.0 개편으로 접근 가능한 데이터가 확대되면 **개인 포트폴리오 전체의 리스크 기여도 시뮬레이션** 구현 기반이 확대.
