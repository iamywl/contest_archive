# FIN:NECT 챌린지 2026 - 프로젝트 작업 파일

## ⚙️ 작업 원칙 (반드시 준수)

### 1. 참고문헌 관리
- **모든 참고문헌·리서치 자료는 `research/` 폴더에 정리**
- 객관적으로 기술 (주관적 해석/각색 금지, 출처 명시 필수)
- 권장 구조:
  - `research/papers/` — 학술 논문 요약 (저자, 연도, 저널, 핵심 결과)
  - `research/services/` — 상용화된 서비스 분석 (서비스명, 제공사, 기능, 수익모델, 한계점)
  - `research/regulations/` — 금융 규제/정책 자료
  - `research/data/` — 시장 데이터, 통계
  - `research/references.md` — 전체 인용 목록 (APA 또는 IEEE 스타일)

### 2. 제안서 작성 스타일
- **논문 형식으로 작성**: 주장에는 근거(인용) 첨부, 추측·과장 금지
- 문제 정의 → 선행 연구/기존 서비스 분석 → 차별점 → 방법론 → 기대 효과 구조
- 모든 수치·주장에는 출처(research/ 내 파일 또는 외부 레퍼런스) 표기

### 3. 아이디어 구상 원칙
- **실제 상용화된 서비스를 기반으로 아이디어 도출**
- 존재하지 않는 가상 사례 금지
- 벤치마크 대상은 `research/services/`에 먼저 조사 후 차별점 도출
- 예시 벤치마크 후보: 토스, 카카오뱅크, 뱅크샐러드, 핀트, Betterment, Wealthfront, Personal Capital, 미래에셋 m-STOCK, 한국투자 KIS, Fidelity Go 등

### 4. 도식·시각자료 디자인 원칙 (모던 미니멀 단일 톤)

**테마 색상은 한 가지로 통일하고 톤만 변형**한다. 다채로운 알록달록한 색상 사용 금지(촌스러움). 정보 위계는 색상 대신 **타이포·여백·보더의 두께**로 구분한다.

#### 4.1 색 팔레트 (slate 단일 톤)

```text
배경       #ffffff
서브 surface #f8fafc  (slate-50, 그룹 구분 시만)
보더 라이트 #e2e8f0  (slate-200, 기본)
보더 미드   #cbd5e1  (slate-300, 강조)
텍스트 주   #0f172a  (slate-900)
텍스트 보조 #475569  (slate-600)
텍스트 흐림 #94a3b8  (slate-400)
강조 (검정) #0f172a  (RiskScope 행 강조 시 다크 배경 + 흰 글씨)
경고 한정   #dc2626  (오직 "extreme/critical"에만, 카드당 최대 1회)
```

- **데이터 시각화도 단일 톤 척도** 사용 (예: 진하기로 양·강도 구분)
- 카테고리 색 코딩(빨강=A, 파랑=B, 초록=C 식) **금지**
- "다채로운 인포그래픽" 스타일 금지 — 비즈니스/금융 맥락에서 신뢰성 ↓

#### 4.2 타이포 위계

```text
H1 도식 제목         28px / weight 700 / slate-900
H2 부제목·캡션       14px / weight 400 / slate-500
섹션 라벨 (LAYER 등) 11px / weight 700 / slate-400 / letter-spacing 1.5-2
카드 제목            14px / weight 700 / slate-900
강조 수치 (KPI)       40-44px / weight 700 / slate-900
본문                 12-13px / weight 400 / slate-700
캡션·각주             10-11px / weight 500 / slate-400
```

#### 4.3 여백·레이아웃

- 외곽 margin: 60-80px
- 카드 padding: 20-24px
- 카드 간 gap: 16-24px
- 헤더 ↔ 본문 공간: 16-32px
- 보더 두께: 1px 기본, 1.5px 최대 강조
- corner radius: 8-12px (모던, 너무 둥글지 않게)

#### 4.4 시각적 처리

- **그림자 사용 금지** (또는 `0 1 2 / 0.04` 극히 미묘하게만)
- 섹션 구분: 1px solid slate-200 라인
- 카테고리 그룹: 점선 1px slate-200, **stroke-dasharray="3 3"**
- 강조: 흰 배경의 카드를 배경 다크 (`#0f172a`) + 흰 글씨로 반전
- 화살표·연결선: stroke 1-1.5px slate-400, 두꺼운 컬러 화살표 금지

#### 4.5 폰트

```text
font-family: -apple-system, "Apple SD Gothic Neo", "Noto Sans KR", sans-serif
```

- macOS·iOS·Windows·Android·Linux에서 모두 한글 깔끔히 렌더링
- letter-spacing: 영문 라벨에 1.5-2 (작은 폰트 가독성)

#### 4.6 SVG 작성 → PNG 변환 워크플로

```bash
# SVG 작성 (figures/svg/)
# viewBox 기반, 1280×720 / 1280×600 / 1400×800 등 16:9 또는 4:3 비율
# rsvg-convert로 PNG 변환 (1600px 폭, 인쇄·슬라이드 모두 적합)
rsvg-convert -w 1600 figures/svg/NN_name.svg -o figures/png/NN_name.png
```

- 변환 도구: `rsvg-convert` (librsvg, macOS Homebrew/conda 모두 지원)
- 출력 폭: 1600px (인쇄 시 충분, 화면 표시도 적정)
- proposal.md 임베드 시 alt text 필수: `![Figure N — 캡션](figures/png/NN_name.png)`

#### 4.7 도식 사용 가이드

| 상황 | 권장 도식 |
|---|---|
| 4개 항목의 인과·중첩 관계 | 중심 노드 + 4개 카드 (SVG 1) |
| 다층 시스템 구조 | 가로 4-5 카드 × 4-5 행 그리드 (SVG 2) |
| 다중 항목 다축 비교 | 모노톤 매트릭스 + RiskScope 행 강조 (SVG 3) |
| 가격·수치 비교표 | 모노톤 표 + 마지막 행 강조 (SVG 4) |
| 시간 축 / 연혁 | 가로 timeline + 점·라벨 (SVG 5) |
| KPI 4종 한눈에 | 4 카드 + 진척률 바 (SVG 6) |

각 도식은 **하나의 메시지**만 전달. 메시지 개수 ≤ 카드 개수.

## 💡 프로젝트 아이디어 요구사항

### 핵심 아이디어
**사용자 맞춤 금융상품 추천 + 미래가치 시나리오 시뮬레이션 서비스**

사용자의 프로필(연령, 소득, 자산, 투자성향, 목표)에 맞춰 금융상품을 추천하고, 해당 상품의 미래가치를 다양한 거시/미시 리스크 시나리오에서 시뮬레이션해 보여준다.

### 핵심 기능

#### 1) 맞춤 금융상품 추천
- 사용자 프로필 기반 개인화 추천
- 상품 유형: 예·적금, 펀드, ETF, 주식, 채권, 연금, 보험 등
- 추천 근거 설명 (설명가능한 AI)

#### 2) 미래가치 시뮬레이션 (차별화 포인트)
단순 수익률 기대치가 아닌 **복합 리스크 시나리오 시뮬레이션**.

**예시: S&P 500 ETF를 담았을 때**
- 📉 **환율 리스크**: 원화가치 폭락/강세 시나리오 → 원화 환산 수익률 변화
- 📉 **기업가치 리스크**: 편입 기업들의 밸류에이션 급락 시나리오
- 📊 **집중 리스크**: 전체 시장 대비 상위 500개 기업 비중 변화 → 패시브 유입/유출 영향
- 📊 **섹터 리스크**: 빅테크 편중(Mag7) → 특정 섹터 충격 시 하락폭
- 📊 **금리 리스크**: 미국 금리 인상/인하 시나리오
- 📊 **지정학 리스크**: 미중 갈등, 전쟁 등 외생 충격
- 📊 **인플레이션 리스크**: 실질 수익률 시뮬레이션

**결과 표현**: 몬테카를로 시뮬레이션 기반 확률 분포, 최악/중위/최선 케이스 시각화

### 시뮬레이션 대상 자산 범위 (확장 가능)
- 해외 ETF (S&P 500, 나스닥 100, 신흥국 등)
- 국내 주식/ETF
- 채권 (국채, 회사채)
- 예·적금, 연금 상품
- 대체자산 (금, 부동산 리츠, 비트코인 등)
