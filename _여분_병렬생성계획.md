# 여분 프로젝트 병렬 생성 계획

> 10개 여분 프로젝트(전국통합데이터 5 + 현대오토에버 5)를 **대회 제출 수준**까지 끌어올리기 위한 병렬 실행 설계.
> 각 폴더 현재: `아이디어.md` ✅ / `제안서.md` ✅. 나머지는 공란.
> 본 문서는 Agent 다중 worktree 동시 실행 설계서이며, 이후 실제 스폰 명령의 입력 브리프로 재사용한다.

## 2. 병렬화 전략

### 2.1 격리 수준
- **Agent 당 1개 worktree** (`isolation: "worktree"`)로 동시 실행해도 충돌 없음.
- 각 worktree는 `main` 기준 분기 → 독립 브랜치에서 커밋 → 최종 `main`으로 fast-forward 병합.
- 10개 동시 실행 시에도 파일 경합 없음(각자 자기 `_여분_*` 폴더만 수정).

### 2.2 트랙 분류
| 트랙 | 프로젝트 | 공통 스택 |
|---|---|---|
| **A: 지도·공공데이터 반응형 웹** (4개) | 실버친구 · 나들이맵 · 런메이트 · 안심귀가 | Next.js 16 + Tailwind v4 + Leaflet + OSM + Vercel |
| **B: 조건 매칭·RAG 웹** (1개) | 청년원스톱 | Next.js 16 + Supabase + pgvector + 법제처 RAG |
| **C: Flutter·온디바이스 PoC** (3개) | 수어톡 · 시각길잡이 · 모두대화 | Streamlit PoC 우선 → Flutter Phase 2 |
| **D: 문서 AI 파이프라인** (1개) | 쉬운말 | Streamlit + OCR + Claude Sonnet |
| **E: 지도·휠체어 그래프** (1개) | 휠체어맵 | Flutter + OSRM self-host 또는 Streamlit 지도 데모 |

트랙별로 **스캐폴드·mock 폴백·공공 API 프록시 패턴**을 재사용해 병렬 비용을 낮춘다.

### 2.3 공유 리소스 (사전 작성 → 전 트랙 참조)
1. **공공 API 프록시 템플릿** (`_여분_공유/lib/public-api-proxy.ts`)
   - 5분 TTL + stale-while-revalidate 유틸
   - 인증키 없을 때 mock fixture 폴백
2. **지도 컴포넌트** (`_여분_공유/components/Map.tsx`)
   - Leaflet + OSM, 마커 클러스터, 팝업 공용
3. **고대비·큰글씨 디자인 토큰** (`_여분_공유/tailwind-a11y.config.ts`)
   - 실버친구·쉬운말·시각길잡이가 재사용
4. **스크린샷 자동화 스크립트** (`_여분_공유/scripts/capture.mjs`)
   - Playwright로 핵심 화면 PNG 저장
5. **프로젝트 CLAUDE.md 템플릿** (`_여분_공유/templates/CLAUDE.md`)
   - 각 폴더의 작업 규칙(문서·코드·캡처 의무) 상속
6. **로컬 LLM 어댑터** (`_여분_공유/lib/local-llm.ts`, `_여분_공유/lib/local_llm.py`) ⭐
   - Ollama REST(`http://localhost:11434`) 래퍼
   - JSON 스키마 강제 (`outlines` 또는 `llama.cpp grammar`)
   - 스트리밍 응답 표준화
   - 모델 alias (`"chat"` → `llama3.3:70b`, `"ko"` → `qwen2.5:32b`, `"vision"` → vision model)
7. **로컬 STT 어댑터** (`_여분_공유/lib/local_stt.py`) ⭐
   - whisper.cpp / MLX Whisper 통합 래퍼
8. **로컬 TTS 어댑터** (`_여분_공유/lib/local_tts.py`) ⭐
   - Kokoro (한국어) 기본, fallback XTTS

> 트랙별 에이전트는 상위 `_여분_공유/`를 **읽기 전용**으로 참조. 수정은 금지.
> ⚠️ **클라우드 AI SDK import 금지** — `anthropic`, `openai`, `@anthropic-ai/sdk` 등 의존성 추가 불가 ([CLAUDE.md §7](./CLAUDE.md)).

## 4. Agent 브리프 템플릿

각 에이전트는 아래 브리프 1장을 받아 **자기 폴더만** 완성한다.

```
역할: 여분 프로젝트 {프로젝트명} 의 MVP 완성
입력:
  - 폴더: _여분_{대회}_{프로젝트명}/
  - 기존: 아이디어.md, 제안서.md (읽기 전용 참조)
  - 상위 CLAUDE.md: {2026_전국 통합데이터 활용 공모전 | 현대오토에버_배리어프리}/CLAUDE.md
  - 공용: _여분_공유/ (읽기 전용 템플릿·유틸)

산출물 (필수, 이 순서):
  1. {폴더}/CLAUDE.md        ← 상위 규격 상속 + 프로젝트별 특이사항
  2. {폴더}/docs/개발계획서.md ← 공용 템플릿 구조, 표 중심
  3. {폴더}/dev(또는 src)/   ← 최소 MVP 스캐폴드 + 빌드 통과 + mock 폴백
  4. 실제 구동 → 스크린샷 자동화 실행 → PNG 저장
  5. {폴더}/docs/개발보고서.md ← 캡처 임베드 + 검토 체크리스트

준수:
  - 각 파일 완성 시 **즉시 커밋** (요소기능 단위)
  - Co-Authored-By: Claude 트레일러 금지
  - 공용 API 키 없을 때도 mock 모드로 구동 가능해야 함
  - 제안서의 스택·화면 구조와 **불일치 금지** (제안서가 규격)

브랜치: feat/여분-{프로젝트명}
완료 시: main으로 rebase-merge 후 보고
```

## 5. 트랙별 공용 작업 (선행 — 1회만)

병렬 실행 전에 **공용 리소스 에이전트 1명**이 먼저 만들고 커밋·푸시한다.

| 작업 | 산출물 | 예상 분량 |
|---|---|---|
| 공용 디렉터리 생성 | `_여분_공유/` | - |
| 공공 API 프록시 템플릿 | `lib/public-api-proxy.ts` + mock fixtures | ~200 LOC |
| 지도 컴포넌트 | `components/Map.tsx` (Leaflet) | ~150 LOC |
| 고대비 a11y 토큰 | `tailwind-a11y.config.ts` | ~50 LOC |
| 스크린샷 자동화 | `scripts/capture.mjs` (Playwright) | ~120 LOC |
| 폴더 CLAUDE.md 템플릿 | `templates/CLAUDE.md` | ~80 LOC |
| 공용 `README.md` | 사용법 | ~40 LOC |

완료 후 **해당 커밋 SHA를 기준 브랜치로 고정**하여 이후 10개 worktree가 이 상태에서 분기.

## 6. 병렬 실행 설계

### 6.1 Phase 0 — 공용 리소스 준비
- Agent × 1 (foreground)
- 예상 소요: 1 세션
- 출력: `_여분_공유/` 전체 + 커밋·푸시

### 6.2 Phase 1 — 10개 프로젝트 동시 생성
- Agent × 10 (병렬, 각자 worktree, background)
- 한 번의 메시지에서 10개 `Agent` 툴콜 동시 실행
- 각 에이전트 독립적으로:
  1. CLAUDE.md 작성 → 커밋
  2. 개발계획서.md 작성 → 커밋
  3. MVP 스캐폴드 작성 → 빌드 통과 → 커밋
  4. 구동·캡처 → 커밋
  5. 개발보고서.md → 커밋
- 예상 소요: Wall-clock 1~2 세션 (병렬이므로 총 10배 아님)

### 6.3 Phase 2 — 통합·병합
- Agent × 1 (foreground)
- 각 브랜치 순차 `git merge --ff-only`
- 전체 `pnpm -r build` 검증 (workspace 있으면)
- README.md 갱신 — 10개 여분 프로젝트 목차 추가
- 단일 푸시로 `origin/main` 갱신

### 6.4 Phase 3 — QA·제출 준비 (선택)
- 경쟁력 높은 상위 2~3개 프로젝트 선별
- `todo/` 폴더에 배포·녹화·서명 체크리스트 추가
- 최종 제출물: 공식 붙임 양식 hwpx/pdf 변환

## 7. 프로젝트별 MVP 스코프 (최소 구동)

실제 빌드·캡처가 가능해야 **개발보고서가 허구가 되지 않는다**. 각 프로젝트 최소 스코프:

| 프로젝트 | MVP 화면 (최소) | 핵심 API 연동 | Mock 폴백 | **로컬 AI** |
|---|---|---|---|---|
| 실버친구 | 홈·쉼터·음성비서 | 쉼터·경로당 프록시 | 서울 시연셋 | **Ollama llama3.3:70b** + outlines |
| 나들이맵 | 홈·지도·AI플랜 | 주차장·화장실·수유실 | 수도권 시연셋 | **Ollama llama3.3:70b** + outlines |
| 런메이트 | 신호등·코스지도·기록 | 대기질·기상·자전거 | 한강공원 시연셋 | 규칙 + **Ollama qwen2.5:32b** |
| 안심귀가 | 홈·경로·보호자공유 | CCTV·가로등·편의점 | 관악 시연셋 | CPTED 가중치 + **qwen2.5:7b** |
| 청년원스톱 | 매칭·상세·캘린더 | 온통청년·고용24 모킹 | 페르소나 3명 | **Ollama devstral-2:123b** + **nomic-embed-text** RAG (FAISS) |
| 수어톡 | 녹화·자막 데모 | MediaPipe 데모 | 30문장 프리셋 | **MediaPipe Holistic** + **whisper.cpp** |
| 시각길잡이 | 카메라 데모·음성 | YOLO 데모 | 점자블록 샘플 | **Ultralytics YOLO v8n (ONNX)** + **whisper.cpp** |
| 쉬운말 | 업로드·변환 데모 | PaddleOCR + 요약 | Mock 문서 3종 | **PaddleOCR** + **Ollama qwen2.5:32b** |

> ⚠️ **모두 로컬 전용.** API 기반 AI (OpenAI/Anthropic/Google/Cohere 등) 사용 금지. [CLAUDE.md §7](./CLAUDE.md) 참조.

## 8. 커밋·푸시 정책 (요소기능 원칙 재확인)

- 각 에이전트는 **파일 1개 또는 논리적 1기능 완성 시 즉시 커밋**.
- 최소 커밋 단위 예:
  - `feat(실버친구): add CLAUDE.md 작업 지침`
  - `feat(실버친구): add 개발계획서 초안 — 표 중심`
  - `feat(실버친구): scaffold Next.js MVP + mock 폴백`
  - `feat(실버친구): add 홈·쉼터·음성비서 3화면`
  - `feat(실버친구): capture 5 screens via Playwright`
  - `docs(실버친구): 개발보고서 작성 + 검토 체크리스트`
- Claude 트레일러 **금지** (지난 rewrite로 정책화).
- 각 브랜치는 푸시 전 `origin/main` 최신 rebase.

## 9. 리스크·완화

| 위험 | 영향 | 완화 |
|---|---|---|
| 병렬 10 agents → 저장소 경합 | 高 | worktree 격리 필수, 공용 디렉터리 읽기 전용 |
| 공공 API 키 없음 → 구동 실패 | 高 | Mock 폴백을 **의무화** (CLAUDE.md 규칙 반영) |
| 빌드 실패 감지 지연 | 中 | 각 에이전트 완료 조건에 `build 통과` 포함 |
| 제안서 ↔ 구현 불일치 | 中 | 제안서를 규격으로 고정, 다를 경우 제안서 수정 커밋 |
| 문서 품질 편차 | 中 | 공용 CLAUDE.md 템플릿·체크리스트 |
| Agent 지연 (1~2개가 느림) | 低 | 병렬이라 전체 지연 아님, 필요 시 재스폰 |
| fintech/web 패턴 혼동 | 低 | 본 여분은 각 폴더 `dev/` 또는 `src/` 로컬 유지, 중첩 repo 금지 |
